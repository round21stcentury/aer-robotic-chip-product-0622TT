# TT 칩 사양서 — `tt_um_reflex_s3` (반사 코어)

> 2026-06-18 기준. 칩 RTL 정독 + 설계 리뷰 정리. 컨텍스트 복구용 단일 레퍼런스.
> 관련: `00_설계_마스터.md`, 각 스텝 폴더 `수행기록.md`, 메모리 [[c-stage-chip-can-gateway]] [[xadc-trigger-plan]].

## 0. 정체
**TinyTapeout 반사 코어.** 평상시 로봇 명령을 CAN으로 **중계(패스스루)**, 위험 시 명령을 끊고 **반사 동작 주입**. CAN 버스의 **유일한 송신자** → "게이트"가 cross-module 신호가 아니라 **칩 내부 mux 한 줄**.

## 1. 외부 핀 (TinyTapeout 표준 24 IO)
| 핀 | 방향 | 용도 |
|---|---|---|
| `ui_in[0]` | 입력 | **DIP(SW0) = estop 트리거** (rule0, 디지털, 항상) |
| `ui_in[1]` | 입력 | danger1 = 소프트 트리거 (PL이 cfg[9]로) → rule1 |
| `ui_in[2]` | 입력 | MCP2515 `INT` |
| `ui_in[3]` | 입력 | MCP2515 `MISO` |
| `ui_in[7]` | 입력 | arm_enable (안전 게이트) |
| `uo_out[0]` | 출력 | valid |
| `uo_out[1]` | 출력 | fire (반사 매칭) |
| `uo_out[4:2]` | 출력 | action_id (1=estop/2=덕포즈/3=움찔) |
| `uo_out[5]` | 출력 | gate_active (정상명령 차단중) |
| `uo_out[7]` | 출력 | heartbeat |
| `uio[0:2]` | 입력 | **SPI 슬레이브** sclk/mosi/csn (PL→칩, 프로그래밍·정상프레임) |
| `uio[3]` | 출력 | SPI 슬레이브 miso |
| `uio[4:6]` | 출력 | **SPI 마스터** sclk/mosi/csn (칩→MCP2515) |

→ SPI 두 개. 한쪽(uio0~3)으로 PL 명령/설정 받고, 다른쪽(uio4~6+ui3)으로 MCP2515 직접 구동.

## 2. "프로그래밍"은 두 가지 (둘 다 startup)
| | A. PL→칩 | B. 칩→MCP |
|---|---|---|
| 대상 | **칩 자신**(규칙·임계·속도·움찔틱·정상프레임) | MCP2515 (CAN 컨트롤러) |
| 트리거 | 칩은 수동(슬레이브). PL(chip_feeder)이 부팅 때 SPI로 써넣음 | 칩 능동. 리셋 직후 **init_start 1클럭 펄스**(자동) |
| 방식 | SPI 슬레이브 레지스터 쓰기 | mcp_init FSM이 SPI 마스터로 |

## 3. A. 칩 프로그래밍 — SPI 슬레이브 (방식 + 레지스터맵)
**프로토콜:** Mode 0, **24비트 트랜잭션** = `[8b 명령]{rw, addr[6:0]}` + `[16b 데이터]`. rw=0 쓰기 / rw=1 읽기. 클럭도메인 2단동기+엣지검출(PL SPI 느슨해도 견고).

| 주소 | 이름 | R/W | 의미 | 리셋값 |
|---|---|---|---|---|
| 0x00 | ID_MAGIC | R | 0xCAFD | — |
| 0x01 | VERSION | R | 0x0531 | — |
| 0x02 | CONTROL | R/W | bit0=enable | 0x0001 |
| 0x03 | SPI_DIV | R/W | 칩→MCP SCLK 반주기(런타임 속도) | 0x0004 |
| **0x10~13** | **RULE0~3** | R/W | 반사 규칙 (§4) | 0x39/0x1A/0/0 |
| **0x18~1B** | **THRESH0~3** | R/W | 규칙별 아날로그 임계 | 0 |
| 0x30 | XADC_VAL | R/W | 현재 아날로그값 (PL이 흘려줌) | 0 |
| 0x40~45 | RECOIL_DELTA_J1~6 | R/W | (RX 기반 현재포즈 recoil용 — parked) | 0 |
| **0x46/0x47** | **FLINCH_LO/HI** | R/W | **움찔(act3) 1회성 지속 = 클럭 틱(32비트)** | 10,000,000 (0.2s@50MHz) |
| **0x48** | **REFLEX_SPEED** | R/W | **반사(덕포즈/움찔)가 주입하는 0x151 속도율(1~100)** = 실로봇 move_spd_rate | 100 (최대) |
| 0x50 | NORM_ID | R/W | 정상 프레임 CAN id | 0 |
| 0x51~54 | NORM_D10/32/54/76 | R/W | 정상 프레임 8바이트 | 0 |
| 0x55 | NORM_SEND | W | 쓰면 정상프레임 송신 펄스 | — |
| 0x20 | STATUS | R | init_done/gate/pose/action_id | — |
| 0x21~28 | MCP 되읽기 | R | canstat/canctrl/cnf1~3/eflg/tec/rec/canintf | — |

## 4. 규칙 인코딩 = 프로그래밍 가능 반사 (핵심)
**RULE 16비트:** `[2:0]=act` (1=estop, 2=덕포즈, 3=움찔) · `[3]=en` · `[5:4]=prio` · `[6]=src` (0=디지털핀, 1=XADC).
**판정:** `danger_raw[i] = src ? (xadc_val ≥ thresh[i]) : danger_pin[i]`. 활성 규칙 중 **최고 prio** 선택.
**danger 핀:** danger[0]=DIP, danger[1]=소프트, danger[2:3]=0.

**트리거 2개 / FSR는 3기능 중 택1:**
| 트리거 | 규칙 | 기능 | 비고 |
|---|---|---|---|
| **DIP(SW0)** | rule0=0x39 | **estop 항상** | 디지털, prio3, 리셋부터 켜짐 |
| 소프트(0x7F0) | rule1=0x1A | 덕포즈 | 디지털 danger[1] |
| **FSR(XADC)** | rule2 | **estop / 덕포즈 / 움찔 중 1** | src=1, 임계 1개(thresh2). PS가 FSR_RULE로 선택 |

FSR_RULE 값: **0x79=estop · 0x5A=덕포즈 · 0x5B=움찔 · 0=비활성.** estop(prio3) > 덕포즈/움찔(prio1).

## 5. 반사 동작 (action_id → CAN 프레임)
| act | 동작 | 프레임 | 해제 |
|---|---|---|---|
| 1 | **estop** | `0x150`, D0=0x01, 주기송신 | 센서-off 즉시 |
| 2 | **덕포즈** | `0x155/156/157` 순환, 데이터=홈(전부 0). 관절=2×부호32비트(0.001°,BE) | 센서-off 즉시(레벨/추종) |
| 3 | **움찔** | 덕포즈와 동일(홈) 프레임 | ★엣지 1회성: 발동 후 **FLINCH_TICKS 자동해제**(센서 계속 눌려도) → 센서 내렸다 올려야 재발동 |

★**덕포즈/움찔은 포즈 프레임 앞에 `0x151`(REFLEX_SPEED 속도율)도 주입** → 실로봇이 그 속도로 홈 이동(마지막 설정 속도 무관). HIL 시뮬은 0x151 속도 무시(호스트 보간)지만 실로봇 대비 항상 주입. estop(0x150)은 정지라 속도 무관.

★**움찔 = 스텝4 우회 설계.** 원래 "현재포즈 RX→현재+델타"는 칩의 MCP RX(arbiter c2)가 공유 SPI 드라이버에서 송신/관측과 경쟁하는 HW 버그(parked). 대신 **덕포즈를 잠깐(0.2s) 발동했다 자동해제** = "홈쪽으로 홱 움찔했다 복귀". 순수 TX → RX 버그 없음. 타이머는 클럭상대 틱(0x46/0x47)이라 칩 클럭 바꿔도 조정만.

## 6. B. MCP startup 프로그래밍 (mcp_init, 8스텝)
리셋 직후 자동: (0)소프트리셋 0xC0 → **(★10ms 발진안정 지연 — 없으면 CNF 씹힘)** → (1~3)CNF1/2/3=00/C0/80(8MHz/1Mbps) → (4)RXB0CTRL=0x60 → (5)CANINTE=0x01 → (6)CANCTRL=0x00(정상모드) → (7)CANSTAT OPMOD=000 폴링. **init_done=1 후에야** 칩 송신.

## 7. 칩→MCP SPI 드라이버 (spi_master_mcp_v2)
6 op: RESET(0xC0)/WRITE(0x02)/READ(0x03)/RTS(0x80|mask)/STATUS(0xA0)/BITMOD(0x05). Mode 0, MSB first, 상승엣지 샘플. half_div=SPI_DIV[7:0] → sclk=clk/(2·half_div), 런타임 변경.

## 8. 자원 공유 (arbiter mcp_arb4)
SPI 드라이버 1개를 4 클라이언트 공유: c0=init, c1=송신, **c2=수신(스텝3·움찔 비활성)**, c3=관측. ★스텝4 RX버그 = c2 켜면 공유 드라이버 경쟁. 움찔 설계는 c2 안 켜서 회피.

## 9. 관측성 (mcp_probe → 0x20/0x21~28)
칩이 MCP 레지스터를 주기적 되읽어 SPI로 노출 → PL → PS 시리얼. C단계 블랙박스 디버깅을 깬 핵심.

## 10. 데이터 경로
- **정상:** PS(eth) → PL(chip_feeder) → SPI슬레이브(NORM 0x50~55) → mux(normal) → mcp_tx_send → MCP → CAN.
- **반사:** 트리거(DIP/XADC) → reflex_core(규칙중재) → reflex_tx(estop/pose) / reflex_pose_gen(홈) → mux(reflex,gate_active) → MCP → CAN.

## 11. PS make 설정 (전부 재합성 없이 — ELF만, `make build .. && make rerun`)
| 항목 | make 변수 | → 칩 |
|---|---|---|
| 칩→MCP SPI 속도 | `SPI_DIV=6` | 0x03 |
| **FSR 반사 선택** | `FSR_RULE=0x79`(estop)/`0x5A`(덕포즈)/`0x5B`(움찔) | rule2(0x12) |
| 임계 전압 | `THRESH_V=0.80` | thresh2(0x1A), ×4096 |
| **움찔 지속** | `FLINCH_MS=200` (+`CLK_MHZ=50`) | 0x46/0x47, =ms×MHz×1000 틱 |
| **반사 이동속도** | `REFLEX_SPEED=100` | 0x48, 1~100 (실로봇 move_spd_rate). estop은 무관 |
예) `make build FSR_RULE=0x5B FLINCH_MS=150 REFLEX_SPEED=80 && make rerun`

## 12. 상태 / 로드맵
- ✅ HW 검증: 패스스루 · DIP estop · 소프트 덕포즈 · FSR(XADC) estop/덕포즈/움찔.
- ★움찔(act3)이 "스텝4 현재포즈 움츠림"을 RX 없이 대체 — 칩+PL 시뮬 PASS.
- parked: RX 기반 현재포즈 recoil(0x40~45 델타 + c2) — "진짜 현재기준" 필요 시 RX 공존버그 해결 후.
- 칩 규모 ~900 FF · ~790 LUT (FPGA OOC 합성 참고. 옛 A단계 272는 더 작던 시절). 24 IO 제약. 핀맵·레지스터맵이 동결 계약. TT 타일 수 정밀값=OpenLane 필요(보류).

## 13. ★부팅 주의 (HIL)
estop 반사가 ACK해줄 로봇 없이 발동하면 칩 MCP 송신실패 → TEC↑ → bus-off(트리거 내려도 안 풀림, MCP 리셋 필요). **HIL은 virtual_robot 먼저 띄우고 → make rerun → 트리거.** (움찔은 1회성이라 spam 없어 이 위험 적음.) DIP(SW0)는 평소 내려둘 것(올리면 estop 항상).
