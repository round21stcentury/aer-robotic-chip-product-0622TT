# 반사(reflex) 연동 트래젝토리 작성 가이드

> 대상: `back_and_forth.py` / `mobius.py` 같은 동작 스크립트를 만드는 사람.
> 목표: FPGA 반사 칩이 발동했을 때(`/reflex_active`)를 알고, **반사가 끝난 뒤 로봇 실제 위치에서 자연스럽게 트래젝토리를 이어/재계획**하기.

---

## 1. 한 줄 요약

평소엔 네 트래젝토리가 로봇을 움직인다. **위험하면 FPGA 칩이 CAN을 가로채(네 명령 무시) 로봇을 안전 자세로 보낸다.** 그동안·직후를 네 코드가 알아야 로봇이 "뚝 끊기고 점프"하지 않는다. 그걸 알려주는 게 **`/reflex_active`** 토픽이다.

---

## 2. 반사가 일어나면 실제로 무슨 일이?

```
평소:  너의 모션 ─/joint_ctrl_single─→ 컨트롤러 → CAN → 로봇
반사:  너의 명령 ──X(칩이 막음)         칩이 반사 프레임 주입 → 로봇이 안전자세로
```

- 반사 트리거(FSR 센서·DIP)는 **하드웨어(칩)가 µs 단위로** 판단한다. ROS는 관여 안 한다.
- **반사 중(`/reflex_active=True`)에는 네가 `/joint_ctrl_single`로 뭘 보내도 무시된다** (칩의 MUX가 정상명령을 끊음).
- 반사가 풀리면(`False`) 칩이 손을 떼고, **로봇은 반사 자세에 멈춰 있다.** 이제 다시 네 명령을 듣는다.

→ **문제:** 반사 끝난 순간 로봇은 네 트래젝토리가 "있어야 한다고 생각하는 위치"가 아니라 **반사 자세**에 있다. 그냥 원래 트래젝토리를 이어 보내면 로봇이 그 자리로 **확 점프**한다(위험·거침).
→ **해결:** 반사가 풀리면 **로봇 실제 현재각(`/joint_states_feedback`)을 읽어, 거기서부터** 트래젝토리를 다시 보간(re-plan)한다.

---

## 3. 토픽 4개만 알면 된다

| 토픽 | 타입 | 방향 | 의미 |
|---|---|---|---|
| **`/reflex_active`** | `std_msgs/Bool` | **구독** | `True`=반사 중(네 명령 막힘) · `False`=정상 |
| **`/joint_states_feedback`** | `sensor_msgs/JointState` | **구독** | 로봇 **실제** 현재 관절각(rad). 반사 끝난 뒤 여기서 재계획 |
| `/joint_ctrl_single` | `sensor_msgs/JointState` | 발행 | 목표 관절각(position=rad 6개). `velocity[6]`=속도율(1~100%) |
| `/enable_flag` | `std_msgs/Bool` | 발행 | 시작 시 `True` 여러 번. **반사 중에도 계속 True 유지**(로봇 enable 풀리면 안 됨) |

---

## 4. 트래젝토리가 해야 할 일 (4단계 상태머신)

| 상태 | 들어가는 조건 | 동작 |
|---|---|---|
| `RUNNING` | 평소 | 트래젝토리 계산·발행 |
| `REFLEX` | `/reflex_active` False→**True** | 칩이 제어 중. **현재각을 계속 발행**(메일박스 warm 유지 → 칩이 손 떼는 순간 옛 명령으로 **튀는 것 방지**). 트래젝토리 시간 `t`는 정지 |
| `SETTLING` | True→**False** | ★`False`=칩이 반사명령 멈춘 것일 뿐 **로봇은 아직 움직일 수 있음**(움츠림은 시간기반 해제). 정지자세를 명령하며 **`/joint_states_feedback`가 멈출 때까지 대기** |
| `RESUMING` | 로봇 정지 확인됨 | 실제 현재각 → 트래젝토리로 **smoothstep 보간**(점프 방지) → `RUNNING` |

> **핵심:** `/reflex_active` 가 `False` 된 순간 = "반사 명령 끝"이지 "로봇 정지"가 **아니다.** **로봇이 실제로 멈춘 걸 피드백으로 확인한 뒤**에 새 트래젝토리를 내린다 (= "반사 트리거 + 반사 동작 완료 후").
> 반사는 짧을 수도(움츠림 ~0.1~0.3초) 길 수도(덕포즈·정지는 누르는 내내) 있는데 이 상태머신이 둘 다 커버한다.

---

## 5. 반사가 끝나면 로봇은 어디 있나? (참고 — 가정 말고 피드백 읽기)

| 반사 종류 | 끝난 뒤 자세 |
|---|---|
| DIP 정지(freeze) | **현재 자세 그대로** 멈춤 |
| 덕포즈(0x5A) | 홈(전부 0) |
| 움츠림_덕포즈(0x5B) | 홈 쪽으로 한 번 움츠렸다 멈춤 |
| 움츠림_현재(0x5C) | **현재 + J5 약 +0.3rad** |

→ 종류마다 다르니 **외우지 말고 `/joint_states_feedback`를 읽어라.** 그게 항상 맞다.

---

## 6. 코드 — 완성 파일 있음, `trajectory()`만 바꿔라

★**완성 템플릿 = [`reflex_aware_template.py`](reflex_aware_template.py)** (이 폴더). 위 4단계(점프 방지·정지 대기·재진입 보간)가 **다 구현돼 있다.** 너는 **`trajectory(self, t)` 함수 하나만** 네 동작으로 바꾸면 된다:

```python
def trajectory(self, t):       # t=초, 반환=6관절 목표각(rad)
    a = 0.6*math.sin(2*math.pi*0.2*t)   # ← 뫼비우스/백앤포스 등 네 궤적으로
    return [a, 0.8, -0.8, 0.0, 0.0, 0.0]
```
```bash
make run MODE=sim APP=motions/reflex_aware_template.py --speed 30
```

아래는 그 파일이 내부적으로 어떻게 도는지 보여주는 골격이다(이해용 — 실제론 위 파일 써라).

```python
#!/usr/bin/env python3
import math, rclpy
from rclpy.node import Node
from sensor_msgs.msg import JointState
from std_msgs.msg import Bool

JOINTS = ["joint1","joint2","joint3","joint4","joint5","joint6"]
HOME   = [0.0, 0.8, -0.8, 0.0, 0.0, 0.0]
HZ     = 50
SPEED  = 30          # 이동 속도율(1~100). 실로봇은 낮게 시작!
RESUME_SEC = 1.5     # 반사 후 현재각→트래젝토리 재진입 보간 시간

class ReflexAwareMotion(Node):
    def __init__(self):
        super().__init__("reflex_aware_motion")
        self.cmd = self.create_publisher(JointState, "/joint_ctrl_single", 10)
        self.en  = self.create_publisher(Bool, "/enable_flag", 10)
        self.create_subscription(Bool, "/reflex_active", self.on_reflex, 10)
        self.create_subscription(JointState, "/joint_states_feedback", self.on_fb, 10)

        self.reflex = False          # 지금 반사 중?
        self.cur = list(HOME)        # 로봇 실제 현재각(피드백)
        self.state = "RUNNING"       # RUNNING / REFLEX / RESUMING
        self.t = 0.0                 # 트래젝토리 내부 시간
        self.resume_t = 0.0          # 재진입 보간 진행
        self.resume_from = list(HOME)

        for _ in range(5):           # 시작 enable
            self.en.publish(Bool(data=True)); 
        self.create_timer(1.0/HZ, self.tick)

    # ── 콜백 ──
    def on_fb(self, msg):            # 로봇 실제 현재각 갱신
        if len(msg.position) >= 6:
            self.cur = list(msg.position[:6])

    def on_reflex(self, msg):
        if msg.data and not self.reflex:        # 정상→반사 (시작)
            self.state = "REFLEX"
        elif (not msg.data) and self.reflex:     # 반사→정상 (종료) → 재계획
            self.resume_from = list(self.cur)    # ★로봇 실제 위치에서 출발
            self.resume_t = 0.0
            self.state = "RESUMING"
        self.reflex = msg.data

    # ── 네 동작: 시간 t(초)에서의 목표 6관절각 ──
    def trajectory(self, t):
        a = 0.6*math.sin(2*math.pi*0.2*t)        # 예시: joint1 흔들기
        return [a, 0.8, -0.8, 0.0, 0.0, 0.0]

    # ── 메인 루프 ──
    def tick(self):
        self.en.publish(Bool(data=True))         # ★enable 은 항상 유지(반사 중에도)
        dt = 1.0/HZ

        if self.state == "REFLEX":
            return                               # ★발행 안 함 — 칩이 로봇 제어 중

        if self.state == "RESUMING":
            self.resume_t += dt
            r = min(1.0, self.resume_t/RESUME_SEC)
            r = r*r*(3-2*r)                       # smoothstep (부드럽게)
            target_now = self.trajectory(self.t)  # 트래젝토리는 멈춰 있던 t 그대로 이어감
            goal = [(1-r)*self.resume_from[i] + r*target_now[i] for i in range(6)]
            if r >= 1.0:
                self.state = "RUNNING"
            self.publish(goal)
            return

        # RUNNING
        self.t += dt
        self.publish(self.trajectory(self.t))

    def publish(self, q6):
        m = JointState()
        m.name = JOINTS
        m.position = [float(x) for x in q6]
        m.velocity = [0.0]*6 + [float(SPEED)]    # ★[6]=속도율. 안 넣으면 100%(위험)
        self.cmd.publish(m)

def main():
    rclpy.init(); n = ReflexAwareMotion()
    try: rclpy.spin(n)
    except KeyboardInterrupt: pass
    n.destroy_node(); rclpy.shutdown()

if __name__ == "__main__":
    main()
```

핵심 3줄:
- `state=="REFLEX"` → **발행 안 함**.
- 반사 종료 콜백 → `resume_from = 현재각(피드백)`, `state="RESUMING"`.
- `RESUMING` → 현재각에서 트래젝토리로 **smoothstep 보간** 후 `RUNNING`.

---

## 7. 주의사항

- **`/enable_flag`는 반사 중에도 계속 True.** 안 그러면 로봇 토크 풀려 처진다.
- **`velocity[6]`(속도율) 꼭 넣기.** 비우면 100%(최대) — 실로봇 위험. 처음엔 20~30.
- **재진입 보간(RESUME_SEC) 충분히.** 짧으면 반사자세→트래젝토리로 빠르게 움직여 거칠다. 1~2초 권장.
- 반사는 **연달아** 날 수 있다(센서 계속 자극). `RESUMING` 중에 또 반사 오면 콜백이 다시 `REFLEX`로 보내니 자연히 처리됨.
- 반사 토픽을 받으려면 PC에서 **`reflex_status_node.py`** 가 떠 있어야 한다(브리지와 같이). `ros2 topic echo /reflex_active`로 확인.
