# 07_zybo_tt_skeleton 작업 로그

TinyTapeout으로 만들 반사 칩의 스켈레톤을 개발하는 폴더입니다. 칩과 똑같은 입출력을 가진 래퍼를 만들어, PL 안에서 먼저 개발·검증하고, 나중에 별도 FPGA와 진짜 칩으로 옮깁니다. 베이스는 `06_zybo_can_pl`(PL에 CAN을 올린 프로젝트)입니다.

## v1 — 1단계 스켈레톤과 토폴로지 검증 (2026-06-15)

**무엇을 만들었나.** 칩의 핀 규격을 가진 껍데기와, 그 안의 최소 로직을 RTL로 만들고 시뮬레이션으로 검증했습니다.

- `rtl/tt_um_reflex.v` — TinyTapeout 제출용 껍데기. 표준 핀(ui_in/uo_out/uio/ena/clk/rst_n)을 가집니다. 핀 배치를 이 단계에서 정했습니다. 위험 입력은 ui_in[0], 안전 인터록은 ui_in[7], 결정 출력은 uo_out에 valid/fire/action_id/estop_level/heartbeat, SPI는 uio[0..3](SCLK/MOSI/CS_n/MISO), uio[7:4]는 텔레메트리와 외부 ADC 직결 자리로 예약했습니다.
- `rtl/reflex_core_tt.v` — 칩 내부 로직. 위험 입력을 동기화·디바운스해서 빠른 병렬 신호로 반사를 내보내는 경로와, SPI로 설정·데이터를 주고받는 경로를 가집니다.
- `rtl/spi_slave.v` — 동기 오버샘플 SPI 슬레이브(Mode 0). SCLK로 클럭하지 않고 칩 clk로 SPI 신호를 2단 동기화해서 처리합니다. 칩 clk가 SCLK의 8배 이상이어야 안전합니다. 1단계 레지스터는 ID_MAGIC(0xCAFD), VERSION, SCRATCH입니다.
- `sim/tb_tt_um_reflex.v`, `sim/run_sim.sh` — PL 역할을 흉내내는 테스트벤치와 실행 스크립트.

**ADC 입력 방식 결정.** 칩은 변환 제어기를 두지 않고, 이미 디지털로 바뀐 센서 값을 받기만 합니다. 빠른 위험 플래그는 직접 입력 핀으로, 여러 비트짜리 포즈와 센서 값은 SPI로 받습니다. 그래서 핀을 적게 씁니다. 운영 중에는 SPI가 센서 값을 계속 흘려보내는 데 쓰이고, 설정은 부팅 때만 합니다. 나중에 칩이 외부 ADC를 직접 읽는 구조로 갈 가능성은 uio[7:4]를 예약해 열어 두었습니다.

**검증 결과(xsim).** 네 가지 모두 통과했습니다.
- SPI로 ID_MAGIC을 읽어 0xCAFD 확인 (연결 정상)
- SPI로 SCRATCH에 0xBEEF 쓰고 되읽어 일치 (쓰기 경로 정상)
- 위험 입력 → fire=1, action_id=1 즉시 출력 (빠른 반사 경로 정상)
- 위험 해제 → fire=0

SPI는 SCLK 주기 80ns(칩 clk 100MHz의 8배 오버샘플)로 돌렸습니다.

**핀맵 확정 (동결, 2026-06-15).** 개발 속도를 위해 24개 신호 핀을 확정했습니다. 입력 ui_in은 [0]=danger_estop(사용), [1]=danger_fast(예약), [2]=can_rx(예약), [3..6]=여유, [7]=arm_enable(사용)입니다. 출력 uo_out은 [0]=valid, [1]=fire, [2..4]=action_id(여기까지 사용), [5]=ps_gate(예약), [6]=can_tx(예약), [7]=heartbeat입니다. 양방향 uio는 [0..3]=SPI(사용), [4..5]=여유, [6..7]=외부 ADC 예약입니다. 합계로 사용 12개, 예약 6개, 여유 6개이고 클럭과 리셋은 따로입니다. 핀맵을 코드에 반영한 뒤 시뮬레이션이 그대로 통과했습니다(uo_out=00000111 = valid·fire·action_id=1, 예약 비트 0).

핵심 결정 두 가지를 함께 정했습니다. 첫째, 첫 칩은 송신기를 하나로 둡니다. 칩은 반사 판단과 포즈 기반 프레임 생성까지 하되 실제 CAN 송신은 PL의 컨트롤러 하나가 맡습니다. 그래야 같은 ID 반사(디폴트 포즈, 현재 포즈 기반)에서 두 노드가 같은 번호를 동시에 보내 생기는 버스 충돌을 원천적으로 피합니다. 둘째, 칩이 CAN을 직접 송신하는 구조는 다음 칩으로 두되, can_tx·can_rx·ps_gate 핀을 지금 예약해 두어 그때 핀맵이 안 바뀌게 했습니다. 현재 포즈는 SPI의 MOSI로 받고 계산 결과는 MISO로 돌려주므로 포즈 전용 핀은 없습니다.

**다음 (v2 예정).** 디폴트 포즈 회귀 반사. 칩은 동작 번호를 내보내고, PL이 그 번호를 받아 기본 자세로 가는 관절 명령을 만듭니다. 슬라이더와 같은 종류의 메시지라 평소 명령을 막는 선택기(먹스)를 PL 쪽에 도입합니다. 그다음 v3에서 현재 포즈 기반 반사 프레임 생성으로 갑니다. 이 두 가지 모두 첫 셔틀 칩에 들어가야 합니다.

**규칙.** 동작이 바뀌는 수정은 이 파일에 새 버전으로 적고, 파일은 덮어쓰지 않고 새 버전 사본으로 둡니다. 시뮬레이션은 iverilog가 없으므로 Vivado xsim을 씁니다.
