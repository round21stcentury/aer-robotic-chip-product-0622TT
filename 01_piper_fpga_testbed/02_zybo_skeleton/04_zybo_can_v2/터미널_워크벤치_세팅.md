# CAN 디버깅 터미널 워크벤치 (5분할) 세팅

> CAN 작업 시작할 때 항상 띄우는 5개 터미널 구성과 명령어 모음.
> 원클릭 실행: 같은 폴더의 **`can_workbench.sh`** (tmux 5분할 자동 구성)

```
┌─────────────────┬─────────────────┐
│ ① candump       │ ② 버스 모니터링  │
│   (수신 감시)    │   (상태/에러)    │
├─────────────────┼─────────────────┤
│ ③ cansend       │ ④ 시리얼1 Zybo  │
│   (송신/기타)    │   (115200)      │
├─────────────────┴─────────────────┤
│ ⑤ 시리얼2 Due / 기타 작업          │
└───────────────────────────────────┘
```

---

## 0. 사전 준비 (1회)

```bash
sudo apt install tmux can-utils picocom    # candump/cansend = can-utils
                                           # (이 머신: picocom·can-utils 있음, tmux 설치 필요)
```

**시리얼 장치 이름 확인** (보드 연결·전원 ON 상태에서):
```bash
ls -l /dev/serial/by-id/        # 어떤 장치가 어느 tty인지 이름으로 식별 (제일 확실)
ls /dev/ttyUSB* /dev/ttyACM*
```
- **Zybo**: FT2232 칩이라 보통 2개 잡힘 — `ttyUSB0`=JTAG(건드리지 말 것), **`ttyUSB1`=UART** ← 이걸 사용
- **Due**: Programming Port = 보통 **`ttyACM0`**
- 권한 에러 시 (재로그인 1회 필요): `sudo usermod -aG dialout $USER`

**can0 초기화** — 속도 바꿀 때마다 down→set→up 3종 세트 (설정은 안 바꾸면 계속 남는다!):
```bash
sudo ip link set can0 down
sudo ip link set can0 type can bitrate 1000000     # 1M / 500000 / 125000
sudo ip link set can0 up
```

## 각 분할창 명령어

### ① candump — 수신 감시 (항상 켜두기)

```bash
candump -ta -c can0                          # 기본: 타임스탬프 + 색상
candump -ta -e can0,0:0,#FFFFFFFF            # 디버깅: 에러 프레임까지 표시 ★막혔을 때 필수★
candump -ta can0,123:7FF                     # 특정 ID(0x123)만 필터
candump -ta -l can0                          # 파일로 로깅 (candump-날짜.log)
```

### ② 버스 모니터링 — 상태/에러카운터 실시간

```bash
watch -n1 "ip -details -statistics link show can0 | grep -E 'can state|bitrate|RX:|TX:' -A1"
```
읽는 법: `can state ERROR-ACTIVE`=건강 / `ERROR-PASSIVE`=문제 누적(down·up으로 리셋) /
`BUS-OFF`=사망(down·up 필수). RX/TX packets 증가 = 실제 송수신 중.

### ③ cansend — 송신 테스트 / 기타 명령

```bash
cansend can0 456#DEADBEEF                    # 기본 테스트 프레임 (Zybo가 RX로 출력)
cansend can0 123#1122334455667788            # 8바이트 프레임
cansend can0 151#0001320000000000            # Piper 스타일 (0x151 MotionCtrl_2)
for i in {1..100}; do cansend can0 456#DEADBEEF; sleep 0.01; done   # 연사 부하 테스트
cangen can0 -g 10 -I 42A -L 8 -v             # 자동 생성기 (10ms 간격)
```

### ④ 시리얼1 — Zybo UART (115200)

```bash
picocom -b 115200 /dev/ttyUSB1               # 종료: Ctrl+A → Ctrl+X
# 대안: screen /dev/ttyUSB1 115200           # 종료: Ctrl+A → k → y
```
※ Vitis 내장 Serial Monitor와 **동시에 열면 충돌** — 한쪽만 사용.

### ⑤ 시리얼2 — Due (115200) / 기타 작업

```bash
picocom -b 115200 /dev/ttyACM0
```
Due 안 쓸 때는 이 창을 빌드/grep/XSA검증 등 잡일용으로.

## tmux 빠른 조작법 (5분할 쓸 때 필요한 것만)

| 키 | 동작 |
|---|---|
| `Ctrl+b` → 화살표 | 분할창 이동 |
| `Ctrl+b` → `z` | 현재 창 전체화면 토글 (로그 자세히 볼 때) |
| `Ctrl+b` → `[` | 스크롤 모드 (q로 나감) |
| `Ctrl+b` → `d` | 세션 분리 (백그라운드 유지) → `tmux attach -t can`으로 복귀 |
| `Ctrl+b` → `x` | 현재 분할창 닫기 |

## 원클릭 실행

```bash
cd 03_zybo_can_v2 && ./can_workbench.sh            # 기본 1Mbps 가정
ZYBO_TTY=/dev/ttyUSB1 DUE_TTY=/dev/ttyACM0 ./can_workbench.sh   # 장치명 다르면 지정
```
①②는 바로 실행되고, ③④⑤는 명령이 **입력만 된 상태**(Enter만 치면 실행)로 준비된다 —
보드가 아직 안 꽂혀 있어도 에러 없이 뜨게 하기 위함.
