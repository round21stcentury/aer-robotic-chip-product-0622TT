# 01_piper_fpga_testbed — 전체 구조 & HILS 실행 흐름

> **한 줄 요약:** AgileX Piper 6-DOF 로봇암을 **Zybo Z7-20 FPGA**로 제어/검증하는 테스트베드.
> PC가 만든 관절명령이 **이더넷→FPGA→CAN**을 거쳐 (시뮬 또는 진짜) 로봇을 움직이는 **HIL(Hardware-In-the-Loop) 루프**를 닫는다.
> 최종 목표(Phase 2)는 PL(FPGA 로직)이 위험 시 µs 단위로 CAN을 가로채는 **하드웨어 반사(reflex)**.

---

## 1. 큰 그림 — 폴더 3개가 하는 일

```
01_piper_fpga_testbed/
├── 01_piper_sdk/       🧠 "두뇌"   ROS2 컨트롤러(piper_sdk)를 담은 Docker 환경
├── 02_zybo_skeleton/   ⚡ "하드웨어" Zybo FPGA Vivado/Vitis 프로젝트 (보드에 올라가는 코드)
└── 03_hils/            🔗 "지휘소"  위 둘 + 팀원 PC브리지를 묶어 HIL 루프를 돌리는 통합 런처
```

| 폴더 | 역할 | 무엇이 들어있나 | 용량 |
|---|---|---|---|
| **01_piper_sdk** | 관절명령을 CAN 프레임으로 만드는 **ROS2 컨트롤러**(piper_ctrl) + Gazebo, Docker 이미지 `piper-hil:humble` | 패치된 `piper_ros` 워크스페이스 | 226M |
| **02_zybo_skeleton** | **보드(FPGA)에 올라가는 펌웨어**. 이더넷 UDP를 받아 실제 CAN으로 송신(lwIP→XCanPs). 어제의 CAN 브링업~오늘의 이더넷 브리지까지 단계별 폴더 | Vivado/Vitis 프로젝트 5개 + 문서 | 511M |
| **03_hils** | 위 둘과 팀원의 **PC 브리지/가상로봇**을 묶어 `make` 한 줄로 HIL을 돌리는 **통합 지휘소** | Makefile + `piper_pc_handoff/` | 1.6M |

## 2. 데이터 흐름 — 명령 한 번이 어떻게 로봇까지 가나

```
 [슬라이더/모션 GUI]                                    ← 사람이 조종 (03_hils)
        │  /joint_ctrl_single (ROS2 topic)
        ▼
 [piper_single_ctrl]  ← piper_sdk 컨트롤러               (01_piper_sdk, Docker 안)
        │  0x155~0x157 CAN 프레임 인코딩
        ▼
   [ vcan0 ]  가상 CAN 버스
        │
        ▼
 [can_udp_bridge.py]  CAN → 13바이트 UDP 패킹            (03_hils/pc_bridge)
        │  UDP :5000  →  192.168.1.10
        ▼
 ╔═══════════════════════╗
 ║  Zybo FPGA (보드)      ║  lwIP로 UDP 수신 → XCanPs로 실제 CAN 송신  (02_zybo_skeleton/05)
 ╚═══════════════════════╝
        │  실제 CAN 버스 (1 Mbps)
        ▼
   [ can0 ]  USB-CAN 어댑터  ── 또는 ── ★진짜 로봇★
        │
        ▼
 [virtual_robot.py] → [Gazebo 3D 팔]                    (03_hils/pc_bridge, HIL일 때만)
        │  피드백 0x2A5~7 → vcan0 → 컨트롤러
        ▼
   (루프 닫힘)
```

**핵심:** `vcan0`(가상) ↔ `can0`(실제 USB-CAN) 사이에 **이더넷+FPGA**가 끼어있는 게 이 프로젝트의 전부다.
PC는 "FPGA가 뭘 하든" 13바이트 UDP 계약만 지키면 되고, FPGA는 그걸 CAN으로 바꿔 흘린다.

## 3. ⭐ HILS 실행 흐름 — `make slider MODE=hil` 했을 때 도는 파일들

```
make slider MODE=hil
└─ 03_hils/Makefile :  slider → run → _run   (docker 권한 자동 가드)
   └─ docker run --network host --ipc host  piper-hil:humble
         bash /pc_bridge/tests/hils_run.sh        ← 컨테이너 진입점 (env MODE=hil APP=slider)
         │
         ├─ ① /dev/shm/fastrtps_* 청소           ← DDS 찌꺼기 제거(컨테이너 root). ★누적 방지★
         │
         └─ ② exec stage3b_slider.sh             ← hil+slider 전용 검증 스크립트로 위임
               │
               ├─ 1. ros2 launch piper_gazebo     → Gazebo + ros2_control 기동
               ├─ 2. arm_controller active 대기   → (timeout 3, 최대 30초, 격리)
               ├─ 3. can_udp_bridge.py            → vcan0 명령을 UDP로 192.168.1.10:5000 송신
               ├─ 4. virtual_robot.py             → can0 읽어 Gazebo 구동 + 피드백 vcan0
               ├─ 5. piper_single_ctrl            → piper_sdk 컨트롤러(vcan0, judge_flag=false)
               └─ 6. joint_slider_gui.py          → 슬라이더 GUI (사람이 조종)
```

| MODE | 위임 스크립트 | FPGA | 끝단 | gazebo/virtual_robot |
|---|---|---|---|---|
| `sim` | (hils_run.sh 인라인) | ❌ mock_fpga | vcan1 | ✅ |
| `hil` | **stage3b_slider.sh** | ✅ 실제 | can0(USB-CAN) | ✅ |
| `robot` | **real_robot_slider.sh** | ✅ 실제 | **진짜 로봇** | ❌ |

> `hils_run.sh`는 **얇은 디스패처**다: `hil+slider→stage3b_slider.sh`, `robot+slider→real_robot_slider.sh`로 위임하고, sim/모션만 직접 처리. (검증된 스크립트를 그대로 exec → 통합으로 깨질 여지 제거.)

자세한 디버깅 이력·실증vsHIL 차이는 **[03_hils/dev-log.md](03_hils/dev-log.md)** 참고.

## 5. 🗑️ 삭제 가능 파일 요약 (상세는 각 폴더 문서)

전부 **빌드하면 다시 생기는 산출물**이라 안전. 소스(.c/.tcl/.py/.xdc/.md/.xpr/.xsa)는 유지.

| 위치 | 삭제 대상 | 회수 용량 |
|---|---|---|
| `02_zybo_skeleton/*` | Vivado `.runs/.gen/.cache/.hw/.ip_user_files`, Vitis `export/_ide/build`, `.Xil`, `hs_err_pid*.log` | **~279M** |
| `01_piper_sdk/.../ros2_ws` | `build/ install/ log/` (colcon), `__pycache__` | ~11M |
| `01_piper_sdk/.../piper_ros/.git` | git 히스토리 (런타임 불필요) | 204M (선택) |
| `03_hils` | `piper_pc_handoff.tar.gz`(이미 풀림), `__pycache__` | 소량 |

> **안전 청소:** Vivado/Vitis 산출물(~279M) + colcon build/install/log(~11M) → 약 **290M** 회수, 재빌드 10~15분.
> **공격적:** `.git`까지 → 약 **494M** 회수(소스는 남고 커밋 이력만 사라짐).

## 6. 빠른 시작 (HIL)

```bash
# 보드 준비
cd 02_zybo_skeleton/05_zybo_can_eth && make program     # FPGA에 앱 올리기

# HIL 실행
cd ../../03_hils
make prep   MODE=hil       # 인터페이스 셋업 (sudo: vcan0 + can0 1Mbps)
make slider MODE=hil       # 슬라이더로 조종 → 실제 FPGA → Gazebo 팔
```

각 폴더 상세는 해당 폴더의 `<폴더이름>.md` 참고.
