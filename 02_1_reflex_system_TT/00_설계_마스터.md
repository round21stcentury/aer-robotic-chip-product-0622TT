# 02_1_reflex_system_TT — 설계 마스터 (단일 진실)

> TT 칩이 MCP2515 로 CAN 을 직접 몰아 반사(estop·덕포즈·움츠림)를 만든다. 평상시 로봇 명령을
> 패스스루, 위험 시 명령을 끊고 반사를 주입한다. **CAN 버스의 유일한 송신자** 라 "게이트"가
> 크로스모듈 신호가 아니라 칩 내부 mux 한 줄.
> 지난 C단계(`02_reflex_system_dev/04_rs_C_chip_can_full`)를 한 번에 다 올려 실물에서 깨졌고 버그
> 위치를 못 좁힌 실패를 교훈 삼아, **설계→HIL 실물검증→증분→검증** 을 스텝마다 반복했다.
> 작성 2026-06-18 · 최신화 2026-06-20(07 보완·06 테이프아웃 점검 반영).
> 관련: 각 스텝 폴더 `수행기록.md`, `칩_사양서.md`, 메모리 [[c-stage-chip-can-gateway]] [[robot-demo-20mhz]] [[tt-partition-plan]].

## 1. 토폴로지 (★HIL 호환 — 절대 깨지면 안 됨)

```
PC ─eth─ Zybo PS ─PL─ SPI ─ TT칩 ─ SPI ─ MCP2515 ─ H/L ─ USB-CAN(can0) ─ PC
         (lwIP)     (마스터)  (슬레이브+      (CAN 1Mbps)        (candump / virtual_robot)
                              MCP 마스터)
```
- 03_hils 는 **설계무관** — 안 건드린다. 프로그래밍·관찰은 각 스텝 폴더에서.
- 정상명령·피드백은 `virtual_robot` 가 can0(물리버스)에 직접. 피드백 0x2A1/0x2A5~7 도 같은
  물리버스로 브로드캐스트 → 칩이 RX 로 받음(05~07 현재포즈 움츠림). [[hil-rx-feedback-bus]]

## 2. PS↔PL↔칩 인터페이스 (★PS 개입 최소 + 런타임 프로그래밍)

PS 는 **AXI GPIO 몇 줄만** 쓰고/읽는다. **SPI 트랜잭션은 전부 PL(chip_feeder)** 이 자동 수행.
재합성 없이 PS 가 GPIO 로 바꾸는 것(= `make build && make rerun`, ELF만):

| GPIO(출력) | 주소 | → 칩 | 내용 |
|---|---|---|---|
| cfg_gpio | 0x41200000 | cfg_in | [7:0]SPI_DIV [8]enable [9]소프트 반사트리거 |
| cmd_lo/hi/id | 0x41210/220/230000 | 메일박스 | 정상명령(릴레이) |
| thr_gpio | 0x41240000 | thresh2 | FSR 임계(THRESH_V×4096) |
| rule_gpio | 0x41250000 | rule2 | FSR 규칙 선택(FSR_RULE) |
| flinch_gpio | 0x41260000 | 0x46/47 | 움츠림 1회성 틱(FLINCH_MS) |
| d5_gpio | 0x41270000 | 0x44 | J5 움츠림 델타(RECOIL_RAD) |
| rspeed_gpio | 0x41280000 | 0x48 | 반사 이동 속도율(REFLEX_SPEED) |
| debounce_gpio | 0x412A0000 | 0x49 | FSR 디바운스 사이클(DEBOUNCE_MS) |
| **hyst_gpio** | **0x412C0000** | **0x4A** | **★07 슈미트 히스테리시스 시프트(HYST_SHIFT)** |
| lat_gpio(입력) | 0x41290000 | lat_decision/issued | 반사지연 측정(트리거→결정·발사 cyc) |
| reflex_gpio(입력) | 0x412B0000 | reflex_active | 반사 활성 1비트(/reflex_active 통지용) |

- **물리 입력 핀**: dip(SW0=G15)=estop, **★07 dip2(SW1=P15)=덕포즈 복귀**.
- **관측성(★핵심)**: 칩 `mcp_probe` 가 MCP 진단 레지스터를 되읽어 → PL → PS EMIO/시리얼. 블랙박스 탈피.

### ★빌드 환경 주의 — 메모리 [[vitis-ascii-path]]
- Vitis 2025.2 는 워크스페이스 경로에 **한글이 있으면 SDT/lopper 빌드가 깨진다.** → Vitis WS 는 ASCII
  `_vitis_ws/sN`. Vivado 합성·xsim·xsct(JTAG) 는 한글경로 정상.
- 입력 axi_gpio·커스텀 AXI 슬레이브도 lopper 가 못 다룸 → PL→PS 관측은 EMIO/단일채널.

## 3. 칩 SPI 슬레이브 레지스터 맵 (07 기준 누적)

| 주소 | 이름 | R/W | 내용 | 리셋값 |
|---|---|---|---|---|
| 0x00 | ID_MAGIC | R | 0xCAFD | — |
| 0x01 | VERSION | R | 0x0542 | — |
| 0x02 | CONTROL | R/W | bit0=글로벌 enable | 0x0001 |
| 0x03 | SPI_DIV | R/W | 칩→MCP SCLK 반주기 | 0x0004 |
| 0x10~13 | RULE0~3 | R/W | [2:0]act [3]en [5:4]prio [6]src(0핀/1XADC) | 0x39 / 0x1C / 0 / **0x2A(07)** |
| 0x18~1B | THRESH0~3 | R/W | 규칙별 아날로그 임계 | 0 |
| 0x30 | XADC_VAL | R/W | PL 이 샘플마다 써넣음 | 0 |
| 0x40~45 | RECOIL_DELTA_J1~6 | R/W | 움츠림 델타(16b 부호). **J5=0x44 사용** | 0 |
| 0x46/47 | FLINCH_LO/HI | R/W | 움츠림 1회성 지속 = 클럭 틱(32b) | 10,000,000 |
| 0x48 | REFLEX_SPEED | R/W | 반사가 주입하는 0x151 속도율(1~100) | 100 |
| 0x49 | DEBOUNCE | R/W | FSR 디바운스 사이클(클럭상대) | 4(sim)/40000(보드) |
| **0x4A** | **HYST** | R/W | **★07 슈미트 히스테리시스 시프트(해제임계=thr-(thr>>n))** | **2(=25%)** |
| 0x50~55 | NORM_ID/D/SEND | R/W | 정상 프레임 릴레이 | 0 |
| 0x20 | STATUS | R | 결정 텔레메트리 | — |
| 0x21~28 | MCP 되읽기 | R | CANSTAT/CNF1~3/EFLG/TEC/REC/CANINTF | — |

규칙 관례: **rule0=DIP estop→freeze(prio3, 항상)**, **rule1=소프트 움츠림_현재(prio1)**,
**rule2=FSR(PS가 FSR_RULE로 선택)**, **★07 rule3=DIP2 덕포즈(prio2)**. 동시다발은 prio 중재가 하나만 고름.

## 4. 칩 핀맵 (07 기준 — 04 v5 동결 호환)

```
ui_in[0]=dip(estop→freeze) [1]=danger1(소프트) [2]=mcp_int [3]=m_miso
        [4]=★dip2(07 덕포즈복귀) [7]=arm_enable                       (5~6 여유)
uo_out[0]=valid [1]=fire [4:2]=action_id [5]=reflex_active(게이트) [6]=mcp_rst(=1상수) [7]=heartbeat
uio[0]=s_sclk(in) [1]=s_mosi(in) [2]=s_csn(in) [3]=s_miso(out)
    [4]=m_sclk(out) [5]=m_mosi(out) [6]=m_csn(out) [7]=여유
```
- danger 매핑: `danger = {dip2, 1'b0, danger1, dip}` → [0]=dip [1]=소프트 [3]=dip2.
- 보드 배선(Pmod JE): JE1=SCK(V12) JE2=SI(W16) JE3=SO(J15) JE4=CS(H15) JE7=INT(V13).
- DIP: SW0(G15)=estop, **★SW1(P15)=덕포즈**. FSR 아날로그: JXADC AD14 = N15(P)/N16(N→GND).
- mcp_rst 는 상수 1(deassert) — MCP 리셋은 SPI 소프트리셋(0xC0)으로 함.

## 5. 반사 동작 (4종, action_id → CAN)

| act | 동작 | 트리거(현재) | 주입 | 해제 |
|---|---|---|---|---|
| 1 | **freeze**(현재포즈 정지) | DIP(SW0) | 정상명령 차단+현재포즈 유지(델타0). ★0x150 estop 안 씀(실Piper=토크해제=처짐) | 센서-off |
| 2 | **덕포즈**(홈, 레벨) | ★DIP2(SW1, 07) / 소프트 | 0x151속도 + 0x155~7=홈, 유지 | 센서-off(레벨홀드) |
| 3 | **움츠림_덕포즈**(홈, 1회성) | FSR(0x5B) | 홈쪽으로 flinch | ★FLINCH_TICKS 자동해제 → 센서 내렸다 올려야 재발동 |
| 4 | **움츠림_현재**(현재+델타, 1회성) | FSR(0x5C) / 소프트 | RX 현재포즈 + J5델타 | ★FLINCH_TICKS 자동해제 |

- 덕포즈/움츠림은 포즈 프레임 앞에 `0x151`(REFLEX_SPEED) 주입 → 실로봇이 그 속도로 이동.
- **★07 슈미트 히스테리시스(칩 결정코어)**: FSR(XADC) 트리거가 발동=thr·해제=thr×(1−1/2^HYST) 밑에서만 →
  꾹 누르는 동안 무필터 FSR 노이즈가 임계 들락날락해도 **1회 락 유지**(06 버그: 재발동·잠깐풀림). HYST 런타임
  프로그래밍(0x4A). PL 레이턴시 트리거도 동일 슈미트(123µs 쓰레기값 제거). [[robot-demo-20mhz]]

|---|---|---|---|
| 0x150 | 비상정지 | B0=0x01 | 칩→버스 (★freeze 설계선 안 씀) |

| 0x155/6/7 | 관절 명령 | 각 관절 int32 BE 0.001° | 칩→버스(포즈 반사) |

## 8. 검증 원칙 (지난 실패 재발 방지)
