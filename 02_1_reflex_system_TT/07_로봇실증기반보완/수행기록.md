# 07_로봇실증기반보완 — 06 + ★슈미트 히스테리시스 (수행기록)

> **06을 복사해 만든 보완판.** 06 기능 전부 동일 + FSR(XADC) 트리거에 **슈미트 히스테리시스**를 칩 결정코어에 추가.

## ⓪ 07 추가분 — 슈미트 히스테리시스 (2026-06-20)
**06 버그:** FSR 아날로그가 무필터로 칩까지 직행(XADC Wizard 평균화 OFF · xadc_reader raw · feeder 1ms 적재). 임계 근처 노이즈/크리프로 `xadc_val` 이 들락날락 → 꾹 눌러도 `danger_stable` 가 2ms↑ 떨어지면 1회성 락(`flinch_done`)이 풀려 **재발동·잠깐풀림 + 덕포즈 끝까지 + 레이턴시 123µs 쓰레기값**. (듀에=DAC 고정전압이라 깨끗 → 2000µs. = 내부버그 아니라 FSR 노이즈+무필터+하드임계.)

**07 해결:**
| 파일 | 변경 | 효과 |
|---|---|---|
| **chip/rtl/reflex_core_c.v** | XADC 소스(src=1) `danger_raw` 를 슈미트로: 발동 `>=thr`, 해제 `<thr-(thr>>2)`(=75%), 사이는 데드밴드 유지. `localparam HYST_SHIFT=2`. | 꾹 누르면 1회 락, **진짜 떼야**(0.75×thr 밑) 재무장. ★`rel_thr` 항상 0<rel<thr → **영구고착 불가** |
| **pl/rtl/reflex_top_s4.v** | 레이턴시 트리거 `trig_raw` 의 FSR 부분도 동일 슈미트(`xadc_hi_lat`). | 한 번 누름=한 번 측정 → 레이턴시 GUI 신뢰 회복(123µs 쓰레기 제거) |
| **chip/sim/tb_hyst_core.v** (신규) | 슈미트 단위시험: ①임계밑 미발동 ②발동1회 ③★꾹누른채 노이즈디더→재발동0 ④★떼면 재무장→재발동(총2회). | `make sim` 에 포함 |

**검증:** `make sim` 3종 전부 PASS — 칩통합 + ★슈미트 4/4 + PL통합. (디지털 DIP 경로 src=0 은 슈미트 미적용 → 기존 그대로, 회귀 없음.)
**TT 영향:** 슈미트는 `reflex_core_c`(칩 결정코어) 안 → **TT 실리콘에 들어가는 로직.** 로봇실증은 FPGA 합성본이라 `make refpga` 로 적용.

### ⓪-b 슈미트 폭 런타임 프로그래밍 (HYST_SHIFT, 2026-06-20)
디바운스와 동일 플러밍으로 **재합성 없이**(make build) 히스테리시스 폭 조절: 칩 `0x4A` 레지스터 `hyst`(시프트값). 해제임계 = `thr-(thr>>hyst)`. **sh=0→1 클램프로 영구락 방지.**
- 칩: spi_slave_full(0x4A reg, 리셋기본2) → tt_um → reflex_core_c(`sh_eff`). PL: chip_feeder_s4(S_HYST 적재) → reflex_top_s4(`hyst_in`, 레이턴시 트리거에도 동일 시프트).
- BD: `hyst_gpio` M12 @0x412C0000(NUM_MI 12→13). PS: `HYST_GPIO`+`HYST_SHIFT`. options.config `HYST_SHIFT=2`(1=50%빡빡/2=25%/3=12.5%헐겁). `make build HYST_SHIFT=1` 일회성.

### ⓪-c DIP2 → 덕포즈 복귀 스위치 (2026-06-20)
지금까지 DIP1(SW0)=estop/freeze 뿐 → **DIP2(SW1) 추가: 켜면 덕포즈(홈) 유지, 끄면 정상 재개**(레벨홀드).
- `spi_slave_full` rule3 리셋기본 `0x0000→0x002A`(src=0·danger[3]·act2 덕포즈·prio2·en). 피더 안 건드림(rule0 estop·rule1 소프트와 동일 방식).
- `tt_um`: `dip2=ui_in[4]` → `danger[3]`. `reflex_top_s4`: dip2 포트 → chip ui_in[4]. BD: dip2 외부포트. XDC: **SW1=P15**(Zybo Z7. 보드 다르면 확인).
- 우선순위: estop(prio3) > DIP2 덕포즈(prio2) > FSR(prio1).

**검증:** ⓪-b/c 반영 후 `make sim` 3종 재PASS(칩통합·슈미트4/4·PL). 칩 tb ui_in[4]=0·PL tb dip2=0/hyst_in=2 라 회귀 없음.
**다음:** `make refpga && make build && make program` → ① FSR 꾹눌러 1회 락 ② HYST_SHIFT 튜닝 ③ DIP2로 덕포즈 복귀 확인.

## (06 원본 기록)

> 05_통합을 **TT칩 클럭 20MHz 조건**으로 (내일 실로봇 실증을 칩과 같은 조건에서) + **반사 지연 측정**(트리거→CAN발사 µs, 시리얼 출력) 추가. 반사 기능은 05와 동일. (05는 50MHz 그대로 둠.)

## ① 20MHz 적용 (클럭 영향받는 것 전부 — 05@50MHz → 06@20MHz)
| 항목 | 05(50MHz) | 06(20MHz) | 이유 |
|---|---|---|---|
| **FCLK_CLK0** | 50MHz | **20MHz** | 칩·AXI·XADC 전부. `add_reflex` `PCW_FPGA0_PERIPHERAL_FREQMHZ {20}`, 실제 `PCW_ACT=20.000000` 확인 |
| SEND/PROBE/SAMPLE_DIV | 50000 | **20000** (1ms 유지) | 클럭상대 스케일(실시간 보존) |
| RESET_DELAY | 500000 | **200000** (10ms 유지) | MCP 발진안정 |
| SPI_HALF (PL↔칩) | 8 (3.125MHz) | **4** (2.5MHz) | 4cyc/half 안전·속도 근접 |
| SPI_DIV (칩→MCP) | 4 (6.25MHz) | **2** (5MHz) | MCP 10MHz 한계 이내·근접 |
| PACE_US | 300 | **400** | 20MHz는 SPI 느려 파이프라인↑ → 여유 |
| CLK_MHZ | 50 | **20** | FLINCH_TICKS 자동재계산 + 지연 µs 환산 |
| Vitis WS | s5 | **s6** | 빌드 격리 |

- RTL 기본값은 `pl/rtl/reflex_top_s4.v` + `chip/rtl/tt_um_reflex_s4.v` 에서 (하위 모듈 SEND/PROBE/RESET 파라미터 전파).
- **불변**(칩클럭 독립): MCP 8MHz 크리스탈·CNF(00/C0/80)·CAN 1Mbps, 이더넷(ENET 별도클럭), 칩 IO·기능.
- ※TT칩 실제 클럭이 20과 다르면 `options.config` CLK_MHZ + RTL DIV들을 그 비율로 조정.

## ② 반사 지연 측정 (트리거 → CAN 발사까지 µs)
- **원리:** `reflex_top_s4`가 칩→MCP SPI(`chip_uio_out[4]sck/[5]mosi/[6]cs`)를 **수동 스니핑** → MCP **RTS 명령(0x80~0x87)** = "프레임을 MCP TXB로 발사" 시점 검출. ★칩(TT설계) 무변경 — 관측만.
- **카운터:** 트리거(dip | 소프트 cfg[9] | xadc≥thr) 상승 → ① reflex_active(결정) → ② 첫 RTS(발사). 클럭사이클 2개 래치(`lat_decision`/`lat_issued`), 트리거마다 재무장.
- **읽기:** AXI GPIO 입력 `lat_gpio @0x41290000` (dual: +0=결정, +8=발사) → PS가 ÷CLK_MHZ → µs, 시리얼 **`[LAT]`** 줄 출력(새 측정 시 즉시).
- **의미:** 트리거→발사(RTS) = **칩 반사 지연**. +CAN 버스 ~120µs(8B@1Mbps) = 버스 도착. 로봇 구동시작(수신)은 하류 → candump/피드백으로 별도.
- **★예상:** FSR 경로는 feeder 샘플(SAMPLE_DIV 1ms)+SEND_DIV(1ms) 포함 → **~2ms**. DIP/소프트는 칩 직결 → ~SEND_DIV(1ms). 실측은 PS `[LAT]` 확인. (더 빠르게 하려면 SEND_DIV/SAMPLE_DIV ↓ — 추후.)

## (이하) 05_통합 상속 — 반사 기능 동일
# 05_통합 — 전 반사 통합 (수행기록)

> step1~4 통합본: **FSR 한 임계로 estop/덕포즈/움츠림_덕포즈/움츠림_현재 택1** + DIP estop 항상 + 모든 포즈반사 속도(0x151).
> step4(`04_현재포즈_움츠림`, 현재포즈 움츠림 + 반사속도) 기반 + step3(`03_덕포즈_반사`, 홈 플린치) 합본.

## FSR 반사 5종 (rule2, 한 임계 thresh2로 택1)
| FSR_RULE | act | 동작 | 해제 |
|---|---|---|---|
| `0x79` | 1 | **estop** (0x150 D0=1) | 센서 뗌 |
| `0x5A` | 2 | **덕포즈** (홈 0x155~7=0) | 누르는 동안 유지(레벨/추종) |
| `0x5B` | 3 | **움츠림_덕포즈** (홈으로 홱 1회성) | 타이머(FLINCH_MS) 자동복귀 |
| `0x5C` | 4 | **움츠림_현재** (현재포즈 + J5 델타 1회성) | 타이머(FLINCH_MS) 자동복귀 |
| `0` | — | 비활성 | — |

+ **DIP(SW0) = estop 항상**(rule0). 소프트(0x7F0, rule1=0x1C) = 움츠림_현재.
+ 모든 포즈반사(덕포즈/움츠림_*)에 **0x151(REFLEX_SPEED) 주입** → 실로봇 이동속도(estop은 정지라 무관).

## 칩 (`tt_um_reflex_s4`, VER 0x0542)
- **`reflex_core_c` 4-action 재설계:** act1 estop(즉시) / act2 덕포즈(레벨=is_home_now, 센서추종) / act3 움츠림_덕포즈(1회성·홈) / act4 움츠림_현재(1회성·현재+델타). 1회성 = flinch_run(flinch_ticks), 모드 = `pose_mode = flinch_run & flinch_recoil`(flinch_recoil=is_rflinch → act4만 현재+델타, act3는 홈).
- **`reflex_pose_gen`:** pose_mode로 홈(0) vs 현재+델타 분기. 0x151 속도(4프레임 사이클 0x151→155→156→157). RX(`mcp_rx_recv` 동기화+R_WAIT_HIGH)로 현재포즈(0x2A5~7).
- act3·act4 둘 다 동일 1회성 타이머(움찔식). **목표만 홈 vs 현재+델타.**

## PL (`reflex_top_s4` + `chip_feeder_s4`)
- feeder: rule2/thr/xadc/recoil_d5/flinch/**rspeed** 프로그래밍. BD `add_reflex_s4` NUM_MI 9 (cfg/cmd×3/thr/rule/flinch/d5@0x41270000/**rspeed@0x41280000**) + XADC VAUX14.
- ★빌드 격리: Vitis WS `_vitis_ws/s5` (04는 s4). Vivado 프로젝트는 폴더 로컬.

## make 옵션 (options.config)
`FSR_RULE`(기본 0x5C 움츠림_현재) / `THRESH_V` / `FLINCH_MS`(1회성 지속) / `CLK_MHZ` / `RECOIL_RAD`(J5 델타 rad) / `REFLEX_SPEED`(이동속도) / `SPI_DIV`.

## 시뮬 ✅ (둘 다 PASS)
- **칩(tb):** 소프트(rule1=0x1C)→act4 현재+델타(J2=45000) · gate · 1회성 자동해제 · 재무장.
- **PL(tb):** FSR `0x5C`→act4 J5=45000(현재30000+델타15000) + **0x151 속도=100** + 1회성. FSR `0x5B`→act3 **홈(0x157=0, 현재포즈 무시)** = ★act3/act4 구분 확인. tx_fail=0.

## 빌드/HW ✅ (2026-06-19)
재합성(WS s5, DRC 0·타이밍 met WNS+4.9ns·6옵션 주입) + program, 보드 건강 TEC=0. ★**HW 실측**: can0에 현재포즈(0x2A7 j5=20000) 주입 + 소프트트리거(0x7F0→rule1=0x1C=act4) → **can0 0x157 J5=37189(현재20000+델타17189) + 0x151 속도=100** = 움츠림_현재+속도 실HW 작동. (부유 XADC 회피 위해 FSR off로 소프트 테스트 — step4와 동일 함정.)
- 다른 FSR 옵션(estop/덕포즈/움츠림_덕포즈)은 칩+PL sim 검증 + step2/step3 HW 재사용. FSR 선택성(rule2)은 사용자 Due로 각 `make build FSR_RULE=..` 테스트.
