# 스텝3 — 덕/홈포즈 반사 (수행기록)

> ★재설계(2026-06-18): 스텝1 패스스루+먹스 토대 위에 스텝2(estop) + ★홈복귀 포즈 반사★.
> 평상시 PC 정상명령 통과 → 위험 시 칩이 정상 끊고 반사 주입:
>   · DIP(SW0) → e-stop(0x150)   · 소프트트리거(0x7F0) → 홈포즈(0x155~7=0)

## 1. 구조 (스텝1 토대 + 반사 체인)
- 토대: spi_slave_full + chip_feeder_s3(relay) + mcp_init/tx_send/probe/arb4/spi_master + mcp_tx_mux + PS lwIP 패스스루.
- 반사: `reflex_core_s3`(rule 기반 estop/pose 결정·우선순위) + `reflex_pose_gen_s3`(홈 0x155~7=0 순환생성) + `reflex_tx_s3`(estop vs pose 선택→tx_*).
- 먹스: `reflex_active = gate_active(estop|pose)`. 평상시 정상, 위험시 반사.
- 트리거: ui_in[0]=DIP(SW0)→rule0(estop), ui_in[1]=소프트(cfg[9])→rule1(pose, 기본 0x001A). PL서 라우팅.

## 2. 시뮬 — ✅ PASS
`chip/sim/tb_tt_um_reflex_s3.v`: ①정상통과 ②pose→홈포즈 주입+정상차단 ③estop(DIP) 우선(>pose) ④해제 재개. rule1=0x001A 확인.

## 3. HIL 실측 — ✅ PASS (자율)
`fpga/tools/hil_pose_test.py`: ①스윕 통과 ②pose 트리거(0x7F0)→can0 0x155~7=홈(joint1=0, 837프레임)+정상차단 ③해제→스윕 재개.

## 4. ★디버깅 (HW 가 시뮬 못 잡은 버그 2개)
**(a) pose_gen 송신 안 됨:** `reflex_pose_gen_s3` 의 `divcnt` 가 16비트(최대 65535)인데 SEND_DIV=100000(>65535) → `divcnt==SEND_DIV-1` 영영 도달X → 홈포즈 펄스 안 나감. 시뮬은 SEND_DIV=2000<65535라 통과(못 잡음). → **divcnt 32비트 + SEND_DIV=50000(<65536 안전망).**
**(b) 수정이 비트에 안 들어감:** RTL 고친 뒤 `build_xsa`(reset_run impl_1만) 재실행 → synth_1 stale 재사용. `reset_run synth_1` 도 ★증분캐시 재사용★으로 부족. **프로젝트 완전 삭제(rm reflex_vivado) 후 재생성** 해야 32비트가 비트에 반영됨. (시리얼 trig=1·TEC=0·can0 무음 = 칩이 송신시도조차 안 함 = pose_gen 미발화 로 진단.)

## 7. ★XADC 아날로그 트리거 추가 (2026-06-18, step3 원설계 완성)
**의도:** step3 원설계 트리거는 FSR(XADC)였고 소프트(0x7F0)는 임시였음. XADC 추가로 완성(+스텝4 재사용).
**칩 무손상:** 칩이 이미 XADC 완비 — `0x30 XADC_VAL`·`0x18~1B THRESH`·`0x10~13 RULE`, `danger_raw[i]=src[i]?(xadc>=thr[i]):danger[i]`. rule0=dip(estop)·rule1=cfg[9](소프트 pose)·**rule2: src=1이면 (xadc>=thr2)**.
**PL 변경:**
 - `chip_feeder_s3.v`: 주기(SAMPLE_DIV)마다 rule_in→rule2(0x12)·thr_in→thresh2(0x1A)·xadc_val→0x30 적재 (st [4:0] 확장, 릴레이 우선 유지). rule1(소프트) 유지 → ★두 트리거 공존★.
 - `reflex_top_s3.v`: +xadc_val/thr_in/rule_in 입력, +reflex_active(=uo_out[5]).
 - `add_reflex_s3.tcl`: NUM_MI 4→6, +thr_gpio(@0x41240000)·rule_gpio(@0x41250000), +xadc_wiz_0(VAUX14 단극 연속 DRP)+xadc_reader+xlconstant(rst), Vaux14 external.
 - `zybo_s3.xdc`: Vaux14_0_v_p=N15(JXADC AD14P)·Vaux14_0_v_n=N16(N→GND).
 - `reflex_s3_main.c`/`build_app_s3.py`: PS가 부팅 시 THR_GPIO=THRESH_CODE·RULE_GPIO=FSR_RULE 기록 → `make build THRESH_CODE=.. FSR_RULE=..` 로 ★재합성없이(~1분) 튜닝★.
**★임계=0x0C29(0.76V):** 실제 아날로그 = `03_hils/analog_input_gen.ino` 가 0.55~1.00V 직결(분압X). 입력 최소 0.55V>0.5V라 0x800은 항상-ON → 0x0C29(05 검증). Due 시리얼에 ≥0.76V 입력시 발동.
**해제:** step3 RX 없음 → 센서뗌 즉시(rule a). (도달기반 해제는 스텝4.)
**시뮬:** `pl/sim/tb_reflex_top_s3.v` ✅ PASS — xadc<임계 무발사, 0x900≥0x800→reflex_active=1+홈포즈(0x157=0) 송신, 센서뗌 해제, tx_fail=0. (run_sim_topc.sh glob 으로 수정: spi_slave_s2→full·mcp_tx_mux 누락.)
**HW:** clean recreate 빌드 + program. 자율검증=패스스루+소프트(0x7F0) 미파괴(hil_pose_test PASS). 보드 건강(CANSTAT=0x00·EFLG=0·TEC=0, step4식 손상 없음 — RX 없으니).
**★XADC 실측 ✅ PASS (2026-06-18, 사용자 Due):** 풀 사이클 확인 — 진행(패스스루)→Due 전압↑(≥0.76V)→★덕/홈포즈 발동·정상차단→전압↓→진행 복구. 실제 아날로그 센서 트리거 실현(원설계 의도 완성). 검증된 v3(소프트만) 비트/소스 백업=`_verified_backup/`.
**튜닝:** `make build THRESH_V=0.65 && make rerun` 로 임계 재합성없이(~1분) 변경. Due 개체 편차로 실제출력 mV 가 0.76V 못 넘으면 임계 낮추면 됨.

## 8. ★움찔(act3) = 스텝4 "현재포즈 움츠림"을 타임드 플린치로 우회 (2026-06-18, 사용자 아이디어, HW PASS)
**문제:** 원래 스텝4(움츠림=현재포즈+델타)는 칩이 현재포즈를 MCP RX로 받아야 함 → RX(arbiter c2) 켜면 공유 SPI 드라이버 경쟁버그(parked, `04_현재포즈_움츠림`).
**우회 아이디어(사용자):** 현재포즈 안 읽고 **덕포즈(홈)를 엣지1회성으로 잠깐 발동했다 자동해제** = "홈쪽으로 홱 움찔". 순수 TX(c2 안 켬) → RX버그 없음.
**칩:** `reflex_core_s3` act3 1회성 — `act3_sel`(rule2=act3 & 트리거) 상승엣지 → `flinch_run` → `fcnt>=flinch_ticks` 만료시 해제+`flinch_done` 래치 → 센서 내려야(`~act3_sel`) 재무장. pose_active=(act2 레벨)|(act3 1회성). + `spi_slave_full` 0x46/0x47 FLINCH_LO/HI(32비트 클럭상대 틱, 리셋기본 10M=0.2s@50MHz) + tt_um 배선.
**PL/PS:** `chip_feeder_s3`가 주기마다 flinch_lo/hi(0x46/0x47) 적재 + `reflex_top_s3` flinch_in + BD `flinch_gpio`(@0x41260000, NUM_MI 7) + PS `FLINCH_TICKS`(GPIO) + Makefile `FLINCH_MS`×`CLK_MHZ`×1000.
**시뮬 ✅:** 칩(tb)=5a 발동→5b flinch_ticks 후 자동해제(xadc 계속 높아도)→5c 센서 내렸다 올리니 재무장+재발동. PL(tb)=FSR 규칙 0x5A(덕포즈)/0x79(estop)/0x5B(움찔) 전환 + 움찔 1회성. (PL sim 타이밍: chip_feeder 전파주기 ~6300클럭 → rule/xadc 변경 후 넉넉히 대기.)
**HW ✅ (FSR_RULE=0x5B 빌드+program):** 보드 건강 **TEC=0** (★estop 빌드 0x79는 TEC=128 bus-off였음 — 움찔은 1회성이라 spam 없어 안전). 자율검증(패스스루+소프트덕포즈) PASS. **사용자 실측 "잘된다"** — FSR 전압↑→홈쪽 0.2초 움찔→자동복귀.
**완성:** FSR가 한 임계(thresh2)로 **estop(0x79)/덕포즈(0x5A)/움찔(0x5B) 택1**, DIP는 estop 항상. → 4스텝 목표가 한 칩에. 칩 FF≈901·LUT≈793(FPGA OOC). 상세 `../칩_사양서.md`·`README.md`. 관련 [[c-stage-chip-can-gateway]].

## 9. ★반사 이동속도 = 0x151 주입 (2026-06-18, 실로봇 대비 Q1 수정, HW PASS)
**문제:** 반사(덕포즈/움찔)가 위치(0x155~7)만 주입 → 실로봇은 **마지막 설정된 move_spd_rate** 로 홈 이동(느릴 수 있음). HIL 시뮬은 호스트보간/Gazebo가 스텝목표를 최대속도로 가서 "빨라 보일" 뿐(시뮬 한정). 코드 근거: `piper_ctrl_single_node.py` MotionCtrl_2(0x01,0x01,vel_all<-clip(velocity[6],1,100)), `wave_v4.py` 주석("0x151 핸들러는 move_spd_rate 버림; 실로봇서만 의미").
**수정:** 반사가 포즈 프레임 앞에 **0x151(ctrl=1·move=1·속도=REFLEX_SPEED) 주입**. `reflex_pose_gen_s3` 사이클 0x151→0x155→0x156→0x157(frame_idx 2비트 wrap). 속도=칩 `REFLEX_SPEED` 레지스터(0x48, 기본 0x64=100). spi_slave 0x48 + chip_feeder 적재 + rspeed_gpio(@0x41270000, NUM_MI 8) + PS/Makefile `REFLEX_SPEED`(make 튜닝). estop(0x150)은 정지라 무관.
**검증:** 칩·PL sim — 덕포즈·움찔 모두 "0x151 속도=100 + 홈포즈(0)" 샘플링 확인. ★HW: 0x7F0 덕포즈 트리거 → can0 에 0x151 D2=100(0x64) 425개 + 0x155 홈 425/425 실측. 보드 TEC=0.
→ 실로봇 반사는 **REFLEX_SPEED(기본 최대100%)** 로 이동 — 마지막 슬라이더 속도 무관. `make build REFLEX_SPEED=80` 로 튜닝.
