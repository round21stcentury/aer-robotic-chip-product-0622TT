# 스텝4 — 현재포즈 움츠림 (수행기록) ★최종 목표

> 스텝3(FSR+홈) 위에 **CAN 수신·디코드 + 현재포즈 기반 움츠림 + 도달판정 해제(rule b)** 를 더한다.
> = "현재 포즈에서 일정 각도만큼 움츠리기" (프로젝트 최종 목표).

## 1. 증분 (스텝3 대비)
- **추가** `mcp_rx_recv.v`(04 검증본) — MCP INT → RXB0 읽기 → rx_id/data (중재기 c2, 이제 활성).
- **추가** `pose_status_decode.v`(04) — 0x2A5~7→현재포즈, 0x2A1 다섯째바이트→도달(reached).
- **교체** 코어 `reflex_core_s3`→`reflex_core_c`(04 풀코어) — estop/홈/움츠림 + pose_mode + ★rule b 해제(도달+센서뗌).
- **교체** 포즈생성 `reflex_pose_gen_s3`→`reflex_pose_gen`(04) — pose_mode=1: 현재+델타(URDF 클램프), 0: 홈.
- **교체** 송신원 `reflex_tx_s3`→`reflex_tx_src`(04, ESTOP_DATA=B0:01).
- **확장** 슬레이브 `spi_slave_s4` = s2 + RECOIL_DELTA_J1~6(0x40~0x45). VERSION=0x0504.
- **추가 PL** `chip_feeder_s4` — 부팅 적재에 델타 J2/J3 추가(규칙 여전히 마지막) + delta GPIO.
- **추가 PL** `reflex_top_s4` — d2_in/d3_in 입력 + delta GPIO.
- 그대로: 드라이버·init·probe·arb4·tx_send·xadc_reader·관측 GPIO·SPI속도 프로그래밍.

## 2. PS↔PL (재합성 없이 PS 변경)
| GPIO | 내용 |
|---|---|
| cfg @0x41200000 | SPI_DIV/enable |
| obs @0x41210000 | MCP 되읽기 관측 |
| thr @0x41220000 | FSR 임계 (THRESH_V) |
| rule@0x41230000 | FSR 규칙 (기본 0x5B 움츠림) |
| delta@0x41240000 | 움츠림 델타 J2(+0) / J3(+8) |
- 적재 순서(PL): SPI_DIV→CONTROL→THRESH→DELTA_J2→DELTA_J3→**RULE(마지막)**. 오발사 함정 차단.

## 3. 시뮬 (xsim) — ✅ 둘 다 PASS
- 칩 `chip/sim/run_sim_top.sh`: VER 0504, 되읽기 CNF, 현재포즈 수신(0x2A5 j2=30000),
  FSR→**0x155 의 J2 = 30000+15000 = 45000**(현재+델타, 디코드 역연산으로 확인), 홈(0) 아님,
  **rule b: 센서만 떼면 유지, 도달(0x2A1) 오면 해제**, tx_fail=0.
- PL `pl/sim/run_sim_topc.sh`: configured, obs0=0x0000C080, 현재포즈 수신→FSR→J2=45000→rule b 해제, tx_fail=0.

## 4. HIL 합격 기준 (★최종)
- `make program` → `make serial`(CNF 관측). PC: `virtual_robot --cmd-iface can0`(피드백 0x2A1/0x2A5~7 을 같은 can0 로 브로드캐스트 — 칩이 RX 로 받음) + 슬라이더로 **임의 포즈**.
- FSR 임계 넘김 → `candump can0` 에 0x155 = **현재 포즈 + 델타**(예 J2 +15도) + **로봇이 그만큼 움츠림**.
  도달 + FSR 뗌 → 풀림. ← ★목표 달성.
- 델타 바꾸기: `make build DELTA_J2=0x4E20 && make rebuild`(재합성 없음). 임계/규칙도 동일.

## 5. ★HIL 주의 (메모리 hil-rx-feedback-bus)
- virtual_robot 이 피드백을 **물리버스(can0)** 로도 브로드캐스트해야 칩 RX 가 현재포즈·도달을 받음.
  `--status-iface` 가 can0 일 때 자동 브로드캐스트(없으면 RX 반사가 굶음). 03_hils 는 안 건드림.

## 7. ★parked RX 버그 잡고 완성 (2026-06-19, "계속해")
**막혔던 진짜 원인(확정):** `mcp_rx_recv` 가 `mcp_int` 를 **동기화 없이 레벨로**(`R_IDLE: if(!mcp_int)`) 보고, RX0IF 클리어(BIT MODIFY) 후 INT 가 high 로 풀리기 **전에** R_IDLE 복귀 → 빈/직전 RXB0 재독 + c2 가 드라이버 계속 점유 → **송신(반사) 굶음.** ★sim 모델 `common/mcp2515_model_v2` 의 `int_n=~CANINTF[0]`(조합=즉시풀림)이라 sim 은 재독이 안 생겨 PASS — 그래서 못 잡았던 것.
**수정(`mcp_rx_recv`):** ① `mcp_int` 2단 동기화(비동기핀 메타안정) ② 읽기 후 **INT high 확인하고서야 재무장**(`R_WAIT_HIGH`, 그동안 `seq_active=0` 으로 드라이버 점유 해제 → 송신/관측 안 굶김). = 표준 레벨-IRQ 재무장 패턴.
**1회성으로 변경(사용자 "움찔처럼"):** `reflex_core_c` act3(움츠림) 해제를 도달기반(rule b)→**`flinch_ticks` 타이머 1회성**. 발동순간 현재포즈 래치+델타 → 타이머 후 자동복귀 → 센서 내려야 재무장. `spi_slave_full` 에 flinch_lo/hi(0x46/47) 추가(step3 포팅, VER 0x0542), tt_um `flinch_ticks` 배선.
**델타=J5만, 프로그램가능:** `reflex_pose_gen` 관절별 델타 그대로 → **recoil_d5(0x44)=J5 델타.** `RECOIL_RAD`(rad)×57296→0.001도. 기본 0.30rad=17189(≈17.2°).
**PL 완성:** step4 feeder/top/BD 가 step1/2 수준(rule/thr/xadc/delta/flinch 프로그래밍·XADC 없음)이라 **step3 완비본 포팅** — `chip_feeder_s4`(rspeed자리→recoil_d5), `reflex_top_s4`(s3→s4·d5_in), `add_reflex_s4`(NUM_MI 8+XADC+thr/rule/flinch/d5 GPIO), PS/build_app/Makefile/options.config(`RECOIL_RAD`, FSR_RULE 기본 0x5B). 원본 `*_pre_port` 백업.
**시뮬 ✅:** 칩(tb)=현재30000+델타15000=45000·1회성 자동해제·재무장. PL(tb)=PS GPIO→feeder→SPI→칩(RX 0x2A7 j5)→0x157 J5=45000·1회성, **풀체인 PASS.**
**HW ✅ (2026-06-19, autonomous):** 재합성(DRC 0·타이밍 met WNS+8.9ns·RECOIL_RAW 17189)+program, 보드 건강 TEC=0. ★can0에 현재포즈(0x2A7 j5=20000) 직접 주입 + 소프트트리거(0x7F0) → **0x157 J5=37189(=현재20000+델타17189) 실측** — 칩이 RX로 현재포즈 읽어 + 델타 더해 움츠림. **RX 버그 해결 HW 확정.**
★**부유 XADC 함정(Due 없는 내 환경 한정):** N15 떠서 FSR(rule2=act3)가 부팅때 거짓발동→1회성 flinch_done 래치→소프트트리거 재무장 막힘. **FSR_RULE=0 끄니 소프트트리거로 J5=37189 정상** = 설계 정상, 부유 XADC만 테스트환경 이슈(step3도 FSR는 Due로 검증). 보드는 FSR_RULE=0x5B 복원.
**사용자 풀 HIL(최종):** `virtual_robot`이 피드백(0x2A5~7)을 **물리버스 can0 로 브로드캐스트**해야 칩 RX가 진짜 현재포즈 받음([[hil-rx-feedback-bus]]) + Due로 FSR 구동(부유 방지). 그러면 임의 자세→FSR→그 자세서 J5 0.30rad 움츠렸다 0.1s 후 복귀. 관련 [[c-stage-chip-can-gateway]].
