# 스텝1 — 정상명령 패스스루 (수행기록)

> ★재정의(2026-06-18, 사용자 정정): 스텝1 = **PC 로봇 정상명령이 FPGA(eth→PS→PL→SPI→칩→MCP)를
> 타고 패스스루 되는가** 검증. + **먹스(게이트) 골격을 스텝1부터** 넣어 스텝2~4 에서 reflex_active
> 만 올리면 정상명령을 끊고 반사 주입. (이전 "DIP→0x150 반사" 는 스텝2 로 이동.)
> 데이터 루프: Gazebo 슬라이더 → 이더넷 UDP(5000) → PS lwIP → ★GPIO 메일박스★ → PL chip_feeder
>   → SPI → 칩 spi_slave_full 정상레지스터(0x50~0x55) → mcp_tx_mux(게이트=0 통과) → mcp_tx_send
>   → MCP2515 → CAN → USB-CAN(can0) → virtual_robot → Gazebo. + MCP 되읽기 관측 + RESET_DELAY(리셋지연).
>
> ★HIL: make program → make can-up → (PC) 03_hils make run MODE=hil + virtual_robot --cmd-iface can0
>   → 슬라이더 움직이면 candump can0 에 0x155~7 + 로봇이 따라 움직임 = 패스스루 성공.
>   make serial 에 MCP CNF=00/C0/80 + relay 카운터 증가 확인.
>
> 구성: 칩 spi_slave_full(정상프레임 0x50~0x55) + mcp_tx_mux + chip_feeder_s1(메일박스→SPI 릴레이)
>   + PS lwIP 브리지(05 재사용, XCanPs 대신 메일박스). 칩+PL 시뮬 PASS, FPGA+lwIP ELF 빌드 완료.
>   ★이전 reset-delay HIL 검증(아래)은 MCP 제어 부분에 그대로 유효 — 그 자산 위에 패스스루 얹음.

## 1. 이 스텝의 범위 (★최소 증분)

| 넣은 것 | 뺀 것(다음 스텝) |
|---|---|
| SPI 마스터 드라이버(런타임 속도) + MCP 초기화 | reflex_core(규칙표·우선순위) → 스텝2 |
| DIP 핀 → 0x150 비상정지 주기 송신 | FSR(XADC) 트리거 → 스텝3 |
| **MCP 레지스터 되읽기 probe → PS 관측** | 홈/움츠림 포즈 생성 → 스텝3·4 |
| **PS 프로그래밍 가능한 칩→MCP SPI 속도** | CAN 수신·디코드(현재포즈) → 스텝4 |

## 2. 토폴로지 (HIL 호환)

```
PC ─eth─ Zybo PS ─AXI GPIO─ PL(reflex_top_s1) ─SPI─ 칩(tt_um_reflex_s1) ─SPI─ MCP2515 ─CAN─ USB-CAN(can0) ─ PC
            │  cfg_gpio(out): [8]enable [7:0]SPI_DIV          │
            └  obs_gpio(in) : {CANSTAT,CNF1,CNF2,CNF3}/{EFLG,TEC,REC,CANINTF}
```
- **PS 개입 최소**: PS 는 cfg 한 줄 쓰고 obs 읽어 출력만. SPI 트랜잭션은 전부 PL(chip_feeder_s1)이.
- **PS 프로그래밍(재합성 없이)**: 칩→MCP SPI 속도 = `make build SPI_DIV=6` (sclk=FCLK/(2·SPI_DIV)).

## 3. RTL 구성

### 칩 (chip/rtl)
- `spi_master_mcp_v2.v` — ★04 대비 단 하나: SCLK 분주를 파라미터→**런타임 입력 half_div**. (PS 가 속도 변경)
- `mcp_init.v` — (04 검증본 그대로) 리셋→CNF1/2/3(00/C0/80)→RXB0CTRL→CANINTE→정상모드→폴.
- `estop_tx_src.v` — DIP 2단동기+디바운스 → init 끝난 뒤 SEND_DIV 주기로 0x150(B0=01) 송신 펄스.
- `mcp_tx_send.v` — (04 검증본 그대로) TXB0 적재 12바이트 + RTS.
- `mcp_probe.v` — ★신규 관측: init 후 주기적으로 MCP 9개 레지스터 READ → 칩 reg. **최저 우선순위**.
- `mcp_arb4.v` — ★4클라이언트 중재(init>tx>rx>probe). rx 는 스텝1 비활성(스텝4에서 활성).
- `spi_slave_s1.v` — (04 v4 검증 타이밍) + SPI_DIV(0x03) 쓰기 + MCP 되읽기 노출(0x21~0x28).
- `tt_um_reflex_s1.v` — 칩 최상위. 핀맵 04 v5 호환.

### PL (pl/rtl)
- `spi_master.v` — (05/04 검증본 그대로) 칩 SPI 슬레이브 대상 Mode-0 마스터(24비트 트랜잭션).
- `chip_feeder_s1.v` — ★신규: 부팅 SPI_DIV·enable 적재 + 주기적 MCP 되읽기 7개 → obs0/obs1 GPIO 노출.
- `reflex_top_s1.v` — 칩+feeder+spi_master 결선, MCP 핀 외부, DIP 직접 핀, cfg/obs GPIO.

### 공용 (../common)
- `mcp2515_model_v2.v` — ★데이터시트 기준 모델(재설계 원칙): CNF 는 **설정모드에서만** 써짐,
  RTS 는 **정상모드에서만** 송신(아니면 tx_fail_cnt↑). → 모드/순서 버그를 시뮬에서 드러냄.

## 4. 시뮬 결과 (xsim)  — ✅ 둘 다 PASS

- **칩 통합** `chip/sim/run_sim_top.sh` (tb_tt_um_reflex_s1):
  - ID_MAGIC=CAFD, VERSION=0501, init_done.
  - **되읽기 CNF1/2/3 = 00/C0/80, CANSTAT OPMOD=000(정상).**
  - DIP ON → 송신 증가, ID=0x150, B0=0x01. DIP OFF → 멈춤.
  - **SPI_DIV 변경(0x03)→되읽기 일치 + 통신 유지 + 송신 동작.**
  - **tx_fail_cnt=0** (설정 순서 정상 — 정상모드 아닐 때 송신 시도 없음).
- **PL 통합** `pl/sim/run_sim_topc.sh` (tb_reflex_top_s1):
  - configured=1, **obs0=0x0000C080**(CANSTAT/CNF), obs1=0(에러 없음).
  - DIP ON → reflex_active=1, 버스 송신 증가, ID=0x150. tx_fail_cnt=0.

`make sim` 으로 한 번에 확인.

## 5. FPGA 빌드 / 프로그래밍 (HIL 직전)

```
make fpga       # project → reflex → xsa  (★Chrome 닫기, -jobs 2 OOM 방지)
make build      # PS 앱 (SPI 속도 바꾸려면 make build SPI_DIV=6)
make program    # 보드에 비트+ps7+elf
make serial     # ★MCP CANSTAT/CNF 되읽기가 시리얼에 뜸
make can-up && make can-mon   # DIP(SW0) 올리면 candump can0 에 150# 주기 송신
```

## 6. HIL 합격 기준 (사용자 검증)

1. `make serial` 에 **CANSTAT OPMOD=000 + CNF=00/C0/80** → 칩이 MCP 를 제대로 설정함 확인.
2. **DIP(SW0) ON → `candump can0` 에 `150#...` 주기 송신, OFF → 멈춤.** ← ★이게 떠야 통과.★
3. 안 뜨면: serial 의 EFLG/TEC/REC 로 "설정 틀렸나 / 송신 틀렸나" 실물에서 직접 구분.
   (TEC 폭증=ACK 못 받음=배선/트랜시버/종단, CNF 틀림=설정 경로 문제 …) → 듀에(due_mcp_test)로 대조.

## 7. 배선 (Zybo Pmod JE ↔ MCP2515 모듈, 3.3V 트랜시버)

| JE 핀 | 신호 | 패키지핀 | MCP 모듈 |
|---|---|---|---|
| JE1 | mcp_sck | V12 | SCK |
| JE2 | mcp_si  | W16 | SI |
| JE3 | mcp_so  | J15 | SO |
| JE4 | mcp_cs  | H15 | CS |
| JE7 | mcp_int | V13 | INT |
| SW0 | dip     | G15 | — (비상정지 트리거) |
- MCP VCC=3.3V, GND 공통, CANH/CANL ↔ USB-CAN(can0) 120Ω 종단.
