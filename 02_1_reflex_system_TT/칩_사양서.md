# TT 칩 사양서 — `tt_um_reflex_s4` (반사 코어)

> 최신화 2026-06-20 (07_로봇실증기반보완 기준: 슈미트 히스테리시스·HYST 런타임·DIP2 덕포즈 반영).
> 칩 RTL 정독 + 설계 리뷰 + OOC 합성 점검 정리. 컨텍스트 복구용 단일 레퍼런스.
> 관련: `00_설계_마스터.md`, 각 스텝 `수행기록.md`, 메모리 [[c-stage-chip-can-gateway]] [[robot-demo-20mhz]].

## 0. 정체
**TinyTapeout 반사 코어.** 평상시 로봇 명령을 CAN으로 **중계(패스스루)**, 위험 시 명령을 끊고
**반사 동작 주입**. CAN 버스의 **유일한 송신자** → "게이트"가 cross-module 신호가 아니라 **칩 내부 mux 한 줄**.
VERSION=0x0542. (s1→s4 진화: RX버그수정·움츠림 1회성타이머·슈미트.)

## 1. 외부 핀 (TinyTapeout 표준 24 IO)
| 핀 | 방향 | 용도 |
|---|---|---|
| `ui_in[0]` | 입력 | **DIP(SW0) = estop→freeze 트리거** (rule0, 디지털, 항상) |
| `ui_in[1]` | 입력 | danger1 = 소프트 트리거 (PL이 cfg[9]로) → rule1 |
| `ui_in[2]` | 입력 | MCP2515 `INT` |
| `ui_in[3]` | 입력 | MCP2515 `MISO` |
| **`ui_in[4]`** | 입력 | **★07 DIP2(SW1) = 덕포즈 복귀 트리거** → danger[3] → rule3 |
| `ui_in[7]` | 입력 | arm_enable (안전 게이트) |
| `uo_out[0]` | 출력 | valid |
| `uo_out[1]` | 출력 | fire (반사 매칭) |
| `uo_out[4:2]` | 출력 | action_id (1=freeze/2=덕포즈/3=움츠림_덕포즈/4=움츠림_현재) |
| `uo_out[5]` | 출력 | reflex_active = gate_active (정상명령 차단중) |
| `uo_out[6]` | 출력 | mcp_rst (상수 1 = deassert. 리셋은 SPI 0xC0로) |
| `uo_out[7]` | 출력 | heartbeat |
| `uio[0:2]` | 입력 | **SPI 슬레이브** sclk/mosi/csn (PL→칩, 프로그래밍·정상프레임) |
| `uio[3]` | 출력 | SPI 슬레이브 miso |
| `uio[4:6]` | 출력 | **SPI 마스터** sclk/mosi/csn (칩→MCP2515) |

→ SPI 두 개. 한쪽(uio0~3)으로 PL 명령/설정 받고, 다른쪽(uio4~6+ui3)으로 MCP2515 직접 구동.
`danger = {dip2, 1'b0, danger1, dip}`.

## 2. "프로그래밍"은 두 가지 (둘 다 startup)
| | A. PL→칩 | B. 칩→MCP |
|---|---|---|
| 대상 | **칩 자신**(규칙·임계·속도·움츠림틱·디바운스·히스테리시스·정상프레임) | MCP2515 |
| 트리거 | 칩은 수동(슬레이브). PL(chip_feeder)이 부팅 때 SPI로 써넣음 | 칩 능동. 리셋 직후 init_start 펄스(자동) |
| 방식 | SPI 슬레이브 레지스터 쓰기 | mcp_init FSM이 SPI 마스터로 |

## 3. A. 칩 프로그래밍 — SPI 슬레이브 (레지스터맵)
**프로토콜:** Mode 0, **24비트 트랜잭션** = `[8b 명령]{rw, addr[6:0]}` + `[16b 데이터]`. rw=0쓰기/1읽기. 2단동기+엣지검출.

| 주소 | 이름 | R/W | 의미 | 리셋값 |
|---|---|---|---|---|
| 0x00 | ID_MAGIC | R | 0xCAFD | — |
| 0x01 | VERSION | R | 0x0542 | — |
| 0x02 | CONTROL | R/W | bit0=enable | 0x0001 |
| 0x03 | SPI_DIV | R/W | 칩→MCP SCLK 반주기 | 0x0004 |
| **0x10~13** | **RULE0~3** | R/W | 반사 규칙 (§4) | 0x39 / 0x1C / 0 / **0x2A** |
| 0x18~1B | THRESH0~3 | R/W | 규칙별 아날로그 임계 | 0 |
| 0x30 | XADC_VAL | R/W | 현재 아날로그값 (PL이 흘려줌) | 0 |
| 0x40~45 | RECOIL_DELTA_J1~6 | R/W | 움츠림_현재 델타. **J5(0x44) 사용** | 0 |
| 0x46/0x47 | FLINCH_LO/HI | R/W | 움츠림 1회성 지속 = 클럭 틱(32비트) | 10,000,000 |
| 0x48 | REFLEX_SPEED | R/W | 반사가 주입하는 0x151 속도율(1~100) | 100 |
| 0x49 | DEBOUNCE | R/W | FSR 디바운스 사이클(클럭상대, 노이즈 자가발동 방지) | 4(sim)/40000(보드) |
| **0x4A** | **HYST** | R/W | **★07 슈미트 히스테리시스 시프트.** 해제임계=thr-(thr>>n). 1=50%/2=25%/3=12.5%. 0→1 클램프(락방지) | **2** |
| 0x50~55 | NORM_ID/D10~76/SEND | R/W | 정상 프레임 릴레이 | 0 |
| 0x20 | STATUS | R | init_done/gate/pose/action_id | — |
| 0x21~28 | MCP 되읽기 | R | canstat/canctrl/cnf1~3/eflg/tec/rec/canintf | — |

## 4. 규칙 인코딩 = 프로그래밍 가능 반사 (핵심)
**RULE 16비트:** `[2:0]=act`(1=freeze/2=덕포즈/3=움츠림_덕포즈/4=움츠림_현재) · `[3]=en` · `[5:4]=prio` · `[6]=src`(0=디지털핀, 1=XADC).
**판정:** `danger_raw[i] = src ? 슈미트(xadc_val, thresh[i], hyst) : danger_pin[i]`. 활성 중 **최고 prio** 선택.
**danger 핀:** danger[0]=DIP(estop), danger[1]=소프트, **danger[3]=DIP2(07 덕포즈)**.

**현재 규칙 배치(리셋 기본 + PS):**
| 트리거 | 규칙 | 기능 | prio |
|---|---|---|---|
| **DIP(SW0)** | rule0=0x39 | **freeze 항상**(디지털) | 3(최고) |
| 소프트(0x7F0) | rule1=0x1C | 움츠림_현재(디지털 danger[1]) | 1 |
| **FSR(XADC)** | rule2 | **PS가 FSR_RULE로 1택** | 1 |
| **★DIP2(SW1)** | rule3=0x2A | **덕포즈(홈·레벨홀드)** (07, 디지털 danger[3]) | 2 |

FSR_RULE 값(PS): **0x79=freeze · 0x5A=덕포즈 · 0x5B=움츠림_덕포즈 · 0x5C=움츠림_현재 · 0=비활성.** 기본 0x5C.
우선순위: **estop/freeze(3) > DIP2 덕포즈(2) > FSR(1).**

## 5. 반사 동작 (action_id → CAN 프레임)
| act | 동작 | 프레임 | 해제 |
|---|---|---|---|
| 1 | **freeze**(현재포즈 정지) | 정상명령 차단 + 현재포즈 유지(델타0, pose_freeze). ★0x150 estop 안 씀(실Piper=토크해제=처짐) | 센서-off |
| 2 | **덕포즈**(홈, 레벨) | `0x151`(속도) + `0x155/6/7`=홈(전부 0). 관절=int32 0.001°BE | 센서-off(레벨홀드) |
| 3 | **움츠림_덕포즈**(홈, 1회성) | 덕포즈와 동일(홈) | ★엣지 1회성: FLINCH_TICKS 자동해제(센서 계속 눌려도) → 센서 내렸다 올려야 재발동 |
| 4 | **움츠림_현재**(현재+델타, 1회성) | **RX 현재포즈 + J5 델타**(`0x44`) | ★FLINCH_TICKS 자동해제 |

★**움츠림_현재(act4)는 RX 현재포즈를 씀** — 05 통합서 칩 MCP RX(0x2A5~7 디코드)가 동작(HIL은 virtual_robot이
물리버스로 피드백 브로드캐스트 → 칩 RX 수신, [[hil-rx-feedback-bus]]). 04 시절의 "RX 공존버그→움찔로 우회"는
05 통합서 해소됨. 1회성 타이머는 클럭상대 틱(0x46/47)이라 칩 클럭 바꿔도 조정만.

★**덕포즈/움츠림은 포즈 앞에 `0x151`(REFLEX_SPEED) 주입** → 실로봇이 그 속도로 이동. estop/freeze는 속도 무관.

## 6. ★07 슈미트 히스테리시스 (칩 결정코어, TT 실리콘行)
06 버그: FSR 아날로그가 **경로 전체 무필터**(XADC Wizard 평균화 OFF·xadc_reader raw·feeder 1ms 적재)로
칩 하드임계까지 직행 → 임계근처 노이즈/크리프로 `xadc_val` 들락날락 → 꾹 눌러도 danger 떨어졌다 올라가
**재발동·잠깐풀림**(1회성 락 무력화) + 레이턴시 123µs 쓰레기값. (듀에=DAC 고정전압은 깨끗 2000µs → 내부버그 아님.)
**해결(07):** XADC 소스(src=1) danger 검출을 **슈미트** — 발동 `≥thr`, 해제 `<thr-(thr>>HYST)`(데드밴드 유지).
`rel_thr` 항상 0<rel<thr → **영구고착 불가**(진짜 떼면 반드시 해제). HYST 런타임(0x4A). PL 레이턴시 트리거에도 동일.

## 7. B. MCP startup 프로그래밍 (mcp_init, 8스텝)
리셋 직후 자동: (0)소프트리셋 0xC0 → **(★10ms 발진안정 지연 — 없으면 CNF 씹힘)** → (1~3)CNF1/2/3=00/C0/80
(8MHz/1Mbps) → (4)RXB0CTRL=0x60 → (5)CANINTE=0x01 → (6)CANCTRL=0x00(정상모드) → (7)CANSTAT OPMOD 폴링.
**init_done=1 후에야** 칩 송신.

## 8. 칩→MCP SPI 드라이버 + 자원공유
- `spi_master_mcp_v2`: 6 op RESET/WRITE/READ/RTS/STATUS/BITMOD. Mode 0, MSB first. half_div=SPI_DIV → 런타임 속도.
- arbiter `mcp_arb4`: SPI 1개를 4 클라이언트 공유 — c0=init, c1=송신, c2=수신(RX, 현재포즈), c3=관측.

## 9. 관측성 (mcp_probe → 0x20/0x21~28)
칩이 MCP 레지스터를 주기적 되읽어 SPI로 노출 → PL → PS 시리얼. C단계 블랙박스 디버깅을 깬 핵심.

## 10. 데이터 경로
- **정상:** PS(eth) → PL(chip_feeder) → SPI슬레이브(NORM 0x50~55) → mux(normal) → mcp_tx_send → MCP → CAN.
- **반사:** 트리거(DIP/DIP2/XADC/소프트) → reflex_core(규칙중재·슈미트) → reflex_tx / reflex_pose_gen → mux(gate) → MCP → CAN.
- **RX:** MCP RX → mcp_rx_recv(0x2A5~7·0x2A1 디코드) → reflex_pose_gen(현재포즈) → 움츠림_현재.

## 11. PS make 설정 (전부 재합성 없이 — `make build .. && make rerun`)
| 항목 | make 변수 | → 칩 |
|---|---|---|
| 칩→MCP SPI 속도 | `SPI_DIV=2` | 0x03 |
| **FSR 반사 선택** | `FSR_RULE=0x5C`(현재)/`0x5B`(덕포즈움츠림)/`0x5A`(덕포즈)/`0x79`(freeze) | rule2 |
| 임계 전압 | `THRESH_V=0.65` | thresh2, ×4096 |
| **움츠림 지속** | `FLINCH_MS=200`(+`CLK_MHZ=20`) | 0x46/47 |
| **반사 이동속도** | `REFLEX_SPEED=100` | 0x48 |
| **FSR 디바운스** | `DEBOUNCE_MS=2` | 0x49 |
| **★슈미트 폭** | `HYST_SHIFT=2`(1=50%/3=12.5%) | 0x4A |
예) `make build FSR_RULE=0x5C HYST_SHIFT=1 DEBOUNCE_MS=2 && make rerun`

## 13. ★부팅 주의 (HIL)
estop/freeze가 ACK해줄 로봇 없이 발동하면 칩 MCP 송신실패 → TEC↑ → bus-off(MCP 리셋 필요).
**HIL은 virtual_robot 먼저 → make rerun → 트리거.** DIP(SW0)는 평소 내려둘 것. 움츠림은 1회성이라 spam 위험 적음.
