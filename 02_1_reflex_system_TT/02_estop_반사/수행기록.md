# 스텝2 — e-stop 반사 (수행기록)

> ★재설계(2026-06-18): 스텝1의 **정상명령 패스스루 + 먹스** 토대 위에 반사를 얹음.
> 평상시 PC 정상명령이 칩을 통과(스텝1) → 위험 시 칩이 **정상명령을 끊고(먹스) 0x150 e-stop 주입**.
> (옛 버전은 반사-only 라 패스스루가 없었음 — HIL 에서 정상명령이 안 흘렀음. 이번에 통합.)

## 1. 구조 (스텝1 토대 + 반사)
- **토대(스텝1 그대로):** spi_slave_full(정상프레임 0x50~55 + 설정) + chip_feeder_s2(정상명령 릴레이) +
  mcp_init/tx_send/probe/arb4/spi_master_mcp_v2 + mcp_tx_mux + PS lwIP 패스스루(PACE_US=300, FWD_IDS 6개).
- **반사 추가:** `reflex_core_s2`(danger[0]=트리거 → estop_active, rule0=0x0039 기본) +
  `reflex_tx_s2`(estop_active 면 0x150[B0=01] 주기 송신, gate_active).
- **먹스 결선:** `reflex_active = gate_active`. 평상시 정상통과, 위험시 정상차단+e-stop. (스텝1 먹스에 반사입력만 연결.)
- **트리거 2경로:** ① 물리 DIP **SW0=G15** ② 소프트 `cfg_in[9]`(PS 제어 UDP 0x7F0, d0=1켜기/0끄기). PL 에서 OR → 칩 ui_in[0].
  → 소프트 트리거로 DIP 없이 자율 HW 테스트 + 원격 발동 가능.

## 2. 시뮬 (xsim) — ✅ 칩 통합 PASS
`chip/sim/tb_tt_um_reflex_s2.v` 4검사 통과:
① 평상시(dip=0) 정상명령 0x155 통과 → ② dip=1 → 0x150(B0=01) 주입 → ③ dip=1 중 정상명령 0x156 ★차단★(먹스 무시) → ④ dip=0 → 0x157 재개. tx_fail_cnt=0.

## 3. HIL 실측 — ✅ PASS (자율 + 물리 DIP)
도구 `fpga/tools/hil_estop_test.py` (UDP 직접 주입 + candump, Gazebo 불필요):
- ① 평상시: can0 = 471·155·156·157 (enable+관절 전부 통과)
- ② 소프트 트리거(0x7F0 d0=1): can0 = **0x150 e-stop만** (정상 0x155 차단)
- ③ 해제(0x7F0 d0=0): can0 = 471·155·156·157 복귀
- + 물리 DIP(SW0)로도 동작 확인(사용자).

★주의: **SW0 가 ON(올림)이면 항상 e-stop** (트리거 상시 1) → 정상 패스스루 안 됨. **평상시 SW0 는 OFF(내림).** (첫 테스트서 SW0 가 올라가 있어 3구간 다 e-stop 나옴 — 내려서 해결.)

## 4. 빌드/프로그래밍
스텝1 동일 — `make fpga`(create_project_s2→add_reflex_s2→build_xsa_s2) → `make build`(lwIP, SPI_DIV/PACE_US 주입) → `make program`.
BD: 메일박스 GPIO(cmd_lo/hi/id @0x41210000~) + cfg_gpio(@0x41200000, [9]=소프트트리거) + dip(SW0) + MCP JE + EMIO 관측. 합성 ~5분.
